# 为开源工作付费其实是件好事 - Jacob Kaplan-Moss

> 来源：[https://jacobian.org/2024/feb/16/paying-maintainers-is-good/](https://jacobian.org/2024/feb/16/paying-maintainers-is-good/)

# 为开源工作付费其实是件好事

*警告：下面是一阵发泄。我现在心情不好，对于控制我的语气并不感兴趣。如果你不想听我长篇大论地批评开源一段时间，请跳过这篇文章。*

* * *

本周早些时候，由于一时的沮丧，我在[Mastodon上写道](https://social.jacobian.org/@jacob/111914179201102152)：

> “我们相信开源应该是可持续的，开源维护者应该得到报酬！”
> 
> Maintainer: *introduces commercial features* “Not like that”
> 
> Maintainer: *works for a large tech co* “Not like that”
> 
> Maintainer: *takes investment* “Not like that”

这段话引起了轻微的病毒式传播，我得到了大量的争论和反对声音。（也有很多支持的声音，这很好。）我认为其中一些反对声音是恶意和不友好的解读，但也有些是出于善意的误解或对我的讽刺言辞的误读。这是我的错，我应该直截了当地说清楚自己的意思，而不是绕圈子。

我的基本立场是，**为开源工作付费是好的**，没有任何例外。**我们需要停止批评那些获得报酬的维护者，开始去赞美他们。** 是的，所有的机制都存在某种程度上的缺陷，但那是因为*世界*本身就是有缺陷的，这不是那些接受报酬的人的错。对已经找到谋生之道的维护者大声呼喊是错误的。

## 为什么这很重要

当然，这是个人问题：我参与开源社区已有二十多年了，结识了许多同事和朋友，他们非常希望能以开源为职业。但大多数人做不到，这很糟糕。那些成功的人最终会收到批评他们财务选择和质疑他们道德的恶意言论。

但问题并不止于此。开源*有益于人类*。说开源是人类作为一个物种最显著的集体成功之一或许有些夸张，但也并非完全没有道理！在开源领域，人类几乎全体一起为一个对所有人都有益的事业而努力。如果没有开源，世界将会比我们现在生活的世界糟糕得多。

因此，我希望那些愿意从事开源工作的人能够做到，并且能够过上舒适的生活，满足他们的基本需求。他们正在为对人类有益的事业做贡献；他们不应该为此而牺牲！

## 定义

讨论的一部分问题是对核心术语的共享协议缺乏共识。因为我在我的原始 toot 中使用了 “开源” 这个术语，回复中的一个主题是人们误解了我所说的 “开源” 的含义（甚至更令人疲惫的是，重新辩论 “开源 vs 自由软件” 的争论）。

如果我要有一半的机会来解释我真正的意思，我需要从定义这些术语对我意味着什么开始：

### “开源” / “自由软件”

请注意使用小写是有意为之。我*不*指的是 OSI 定义的 Open Source™，也不是 FSF 定义的 Free Software™。我指的是在最广泛、最包容的意义上的这些术语：“我可以阅读、修改和发布其变体的源代码软件，或许在某些条件下。” 因此，我包括 OSI 和 FSF 的许可证，但也包括[Polyform 许可证](https://polyformproject.org/licenses/) 和 [JSON 许可证](https://www.json.org/license.html)，以及是的 [BSL](https://mariadb.com/bsl-faq-mariadb/) 在我版本的 “开源” 中。）

或许这是一个次要的观点，但“开源”的“极简主义”定义意味着“只有 OSI 批准的许可证” — 或者更糟，“GPL 是唯一的 ‘真正’ 自由软件许可证” — 在这里是问题的一部分。我希望看到在许可选项中更多的实验和多样性，如果这意味着引入一些超出 “任何人都可以用于任何目的” 的额外限制，我对此表示非常认同。在我看来，从[Blue Oak](https://blueoakcouncil.org/license/1.0.0) 到 [BSL](https://mariadb.com/bsl-faq-mariadb/) （甚至比 BSL 更严格的许可证）的广泛许可证“算作” 开源。

因此，在这篇文章中，我将使用 “开源” 来指代从 “完全无阻碍” 到 “带有一些限制” 的这个光谱上的任何内容，并且我不会详细说明 “一些” 限制多少才能被认为仍然是自由的。我甚至会交替使用 “开源” 和 “自由软件” 这两个术语，只是为了强调在这种情况下，这些术语的精确定义对我并不重要。

这种立场可能确实会惹恼一些人，这在某种程度上是有意为之。不是因为我想故意惹恼任何人，但 &mldr; 我这么说吧：如果在“讨论人们如何谋生”的情境下，我对这些术语的马虎使用让你感到不安，那意味着你更在意术语和定义，而不是关心人们，我希望你能在这种不适中停留一会儿。

### “可持续性”

接下来，当我谈论开源中的 “可持续性” 时，我指的是什么？人们使用这个术语表示很多东西 —— 良好的治理、健康的社区、资金支持等等。

当我谈论“可持续性”时，我指的是非常具体的事情：“维护者能否通过编写自由软件过上体面或舒适的生活方式？” 如果开源软件“可持续”，对我而言，这意味着人们可以选择把编写开源软件作为自己的工作，并确保他们至少能够满足基本需求——住房、食物、医疗保健等等。理想情况下，更甚于此；如果编写开源软件能够让人们过上舒适甚至奢华的生活，那将是一件非常棒的事情。

## 开源并不可持续

几乎没有人靠编写自由软件谋生。作为所有软件工程师的百分比，能靠这个谋生的人几乎可以算作零。

当然，有一些公司雇佣人们来从事开源工作：Canonical、Red Hat、Hashicorp 和 Mozilla 是我能想到的几个；我相信你还能列举更多。但是，（a）与写专有软件的数百万工程师相比，这些公司雇佣的工程师数量寥寥无几；（b）这些组织中的几乎所有工程师并非只从事开源工作；几乎所有这些组织都有其商业模式依赖于某些产品的专有部分。因此，即使是最大的开源成功案例也只代表了一个很小的比例。

更贴近现实的是，让我们看看 Django。据我估计，数十万工程师每天都在使用 Django。但有多少人真正获得报酬来直接为 Django 本身工作呢？**一个半**—— DSF 雇佣了一个全职和一个兼职的成员。这就是唯一靠编写 Django 获得报酬的人。Python 本身的情况也类似：数百万人每天使用 Python，但只有不到十几个人是靠此谋生的^。

“可持续性”意味着大约有十几个人能够全职工作在 Django 上，并且获得接近行业中位数的收入。这意味着有*数十*个人全职从事 Python 工作。天哪，仅仅是[PyPI](https://pypi.org/)本身，考虑到其范围、规模和重要性，最起码也应该有一个由10到15人组成的团队。

更重要的是，在开源中，“可持续性”意味着像[“内布拉斯加州的随机人”](https://xkcd.com/2347/)这样维护关键依赖项的人能够生活得很好，并且有几个同事，这样他们就可以时不时地休假。

我们生活的世界与此相去甚远。

## 梦想：全自动豪华同性空间共产主义

这通常是有人挖苦地指出的时候：“你并不是真的讨厌开源，你讨厌资本主义。” 这是一个真实但并不有帮助的声明。

是的，人们不得不在编写开源软件和支付体面医疗保健之间做选择，这是深深植根于我们当前的零和资本主义中的问题，而且根本不是可以归咎于自由软件运动的问题。梦想是社会和政府将公认自由软件确实是公共利益，并适当地资助它。还要解决医疗保健、住房、公共交通和社会安全网等问题，等等等等&mldr;

我绝对百万分之一地支持这个愿景，但这事不会一夜之间发生。实际上，如果发生了，我甚至怀疑在我们的一生中是否会发生。

我们不得不接受现实世界，即使这不是我们想要的世界。这意味着我们必须认同维护人员需要得到报酬的观念。我太经常看到这样的争论： “维护者不应该由私人公司支付，因为政府应该支持他们。” 当然，这听起来很好 - 但*政府没有这样做*！所以这个论点归结为“不应该向开源维护者支付报酬”。我不能接受这一点。

## 每次有人因编写开源软件而得到报酬都是一种胜利。

就在现实世界中，**开源软件的可持续性意味着支付维护者 - 我们应该庆祝每一次这种事情发生！** 每当维护者找到获得报酬的方式，这就是一种胜利。

受雇于 Microsoft 从事 Python 开发？**赢**。

获得资助？**赢**。

在 Patreon 达到可持续的资金水平？**赢**。

获得风险投资来开发自由软件？**赢**。

基于开核心模型建立可持续的业务？**赢**。

Hashicorp？**赢**。

支持一个使用付费托管选项的开放项目？**赢**。

成功使用非-OSI批准的许可证以免受亚马逊效应？**赢**。

**直到我们拥有完全自动化的奢侈同性恋太空共产主义，每一个找到写免费软件并能支付房租的人都代表一种胜利，我们应该相应地庆祝。**

## 相反，批评。

但这不是发生的事情。相反，每当维护者找到获得报酬的方式时，总会有人出现批评和抱怨。非-OSI 许可证“不算”作为开源。受雇于 Microsoft 的人“受到公司利益的束缚”并且不值得信任。Patreon 是“要讨钱”。通过 GitHub 赞助筹集资金意味着“支持 Microsoft 的寻租行为”。风险投资意味着我们被“敷衍了事”或者“被恶心化”。开核心是“诱饵和转换”。

这一切都不是假设的；每个例子都是我所见过的维护者为自己的工作获得报酬时说的实际事情。甚至有一个维护者告诉我他因出售 T 恤而受到批评！

看，我们支持维护者的每一种策略都存在问题。VC 投资确实带来了随之而来的问题。Patreon 或 GitHub（以及 Stripe）从赞助款中抽取一部分也很让人沮丧。PolyForm 或 BSL 引入的额外限制确实违背了[自由软件定义](https://en.wikipedia.org/wiki/The_Free_Software_Definition)的理念。我自己经常因为发现某些我想要的开核工具的关键功能仅限于付费许可而感到沮丧。

**但你仍然可以批评这些体系，同时支持和庆祝维护者！** 尽管你可以尽情地批评 A16Z，我并不在乎（他们也不在乎）。但因为维护者接受了 VC 的资金而对其大发雷霆，这种行为方向错误。导致所有这些不同资金模式存在问题的结构性和社会性问题并非是那些试图以开源谋生的人的错。

在这种情况下，就像在 Dollar General 购物时责备某人一样，尽管 Dollar General 的掠夺性商业模式确实很糟糕，以及导致食物荒漠的政府政策同样令人不满，但这一切并不应归咎于需要牛奶而没有其他选择的个人。

## 纯洁性只会限制开源对社会的价值。

**很多很多人都应该因为编写自由软件而获得报酬，但为了实现这一点，我们必须接受不完美或不纯的机制。** 如果你愿意，可以批评这些机制。请努力改变潜在的社会不公。

但当维护者找到了获取报酬的方法，请为他们庆祝。这对我们所有人都是胜利。
