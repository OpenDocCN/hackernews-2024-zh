# Epic Games CEO暗示，出于反竞争的原因，Apple在欧盟破坏了iPhone的Web应用程序 | TechCrunch

> 来源：[https://techcrunch.com/2024/02/16/epic-games-ceo-suggests-apple-broke-iphone-web-apps-in-the-eu-for-anticompetitive-reasons/](https://techcrunch.com/2024/02/16/epic-games-ceo-suggests-apple-broke-iphone-web-apps-in-the-eu-for-anticompetitive-reasons/)

昨天，Apple确认 [由于其](https://techcrunch.com/2024/02/15/apple-confirms-its-breaking-iphone-web-apps-in-the-eu-on-purpose/) 遵守欧盟数字市场法（DMA），它正故意为欧盟的客户[破坏Web应用程序](https://techcrunch.com/2024/01/25/apple-dma-changes/)。 Epic Games CEO Tim Sweeney [在X的一篇文章中暗示](https://twitter.com/TimSweeneyEpic/status/1758267492781199501) Apple此次决定背后可能有另一个原因：iPhone的Web应用程序对Apple没有经济利益。 虽然Sweeney的公司因App Store费用问题起诉Apple，他显然是这个问题上的一个有偏见的消息源，但他提出了一个人们心中都有的问题。 Apple是因为要保护客户免受来自第三方浏览器引擎的安全风险，正如其所称，破坏iPhone的Web应用程序，还是这个决定更多地是为了扼杀对Apple业务的潜在威胁？

Apple是否会为了保护其收入而降低iPhone上的消费者体验，换句话说？

周四，iPhone制造商发布了一则更新说明其[在欧盟的DMA相关变更](https://developer.apple.com/support/dma-and-apps-in-the-eu/)以解决此问题，此前发现iPhone的Web应用程序（也称为渐进式Web应用程序或PWA）在最近的iOS测试版中在欧盟不再可用。最初，人们担心这些问题只是测试版的错误，但很快，Apple便驳回了这一理论。

在其网站上，[Apple解释道](https://developer.apple.com/support/dma-and-apps-in-the-eu/)，为了遵守DMA，它被迫支持除了Safari使用的WebKit之外的其他Web浏览器引擎。iOS主屏幕Web应用程序一直依赖于WebKit及其安全架构，以保护用户免受在线威胁的影响。这包括存储的隔离和执行“访问影响隐私的能力的系统提示”，Apple表示。

该公司指出，如果没有这种隔离和执行，恶意Web应用程序可能会读取其他应用程序的数据，并获得用户摄像头、麦克风或位置访问权限。由于苹果被迫允许通过DMA的要求使用替代浏览器引擎，该公司选择不让用户面临风险，而是在欧盟地区降低了iOS上的Web应用程序体验。现在，Web应用程序将仅作为网站书签运行，不支持本地存储、徽章、通知和专用窗口。

虽然Sweeney显然对苹果有意见，但他的说法可能有些道理。在苹果解释为何停止在欧盟支持Web应用程序的说明中，该公司承认安全问题有技术解决方案——但它选择不实施。

苹果写道（我们的强调）：

> 解决使用替代浏览器引擎的Web应用程序所涉及的复杂安全和隐私问题**需要构建一种全新的集成架构**，而iOS中目前不存在这样的架构，也不切实际，鉴于DMA的其他要求以及Home Screen Web应用程序的用户采纳率极低。

简而言之，苹果表示它知道如何解决问题，但由于必须遵守DMA（它指出需要“超过600个新API和各种开发者工具”），它决定跳过修复这个问题。

尽管构建“一种全新的集成架构”可能并非易事，但苹果并不意味着对DMA感到惊讶，这项[法规](https://en.wikipedia.org/wiki/Digital_Markets_Act)已经酝酿多年。它有时间为此做准备。为了进一步转移责任，苹果表示，考虑到“Home Screen Web应用程序”的用户采纳率低，人们不会介意它破坏了这一点。

但苹果公司自己的举动与这一解释相矛盾。如果说有什么变化的话，苹果多年来一直在努力使PWA变得更加实用，添加了[功能](https://love2dev.com/blog/apple-ships-service-workers/)，使Web应用程序能够更像本地应用程序，并且可以在其App Store之外轻松分发。与此同时，用户采纳率一直在增长，而非减少。分析师估计，PWA市场到2027年将达到104.4亿美元，年复合增长率为31.9%。

正如Sweeney所主张的那样，替代浏览器引擎完全有可能使PWA变得更加实用，这对苹果的App Store业务构成威胁，因为Web应用程序现在几乎与本地应用程序一样功能强大。

苹果已被要求就其围绕PWA的决定发表评论，但它只在其DMA网站上发布了一份解释作为其回应。
