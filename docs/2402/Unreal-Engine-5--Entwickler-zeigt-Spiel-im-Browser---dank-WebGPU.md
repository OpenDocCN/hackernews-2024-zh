# Unreal Engine 5：开发者展示在浏览器中的游戏 - 多亏了 WebGPU

> 来源：[https://www.pcgameshardware.de/Unreal-Engine-Software-239301/News/5-Entwickler-zeigt-Spiel-im-Browser-dank-WebGPU-1440861/](https://www.pcgameshardware.de/Unreal-Engine-Software-239301/News/5-Entwickler-zeigt-Spiel-im-Browser-dank-WebGPU-1440861/)

当今，管理自己的游戏收藏通常通过一个启动器进行，但如果要玩游戏，首先需要在后台下载和安装。就像以前的 Flash 游戏一样，有些游戏可以直接在 Web 浏览器中运行。未来，甚至可以使用 Unreal Engine 5 制作相应的游戏。

## Unreal Engine 5 在浏览器中

Wonder Interactive 至少在努力：根据该公司的说法，他们计划通过合适的软件，将 Unreal Engine 制作的游戏导出至浏览器进行播放。据称，从 2022 年起，他们已经能够将游戏导出到[较旧的 Web-API *WebGL*](https://theimmersiveweb.com/blog)。现在，根据该公司创始人 Alex St. Louis 的说法，他们似乎已经成功地将游戏导出至新的 *WebGPU*-API。为此，他们发布了一张图片，展示了汽车内部和遥远的风景。

推荐的编辑内容：在这里，您可以看到来自 [PLATTFORM] 的外部内容。为了保护您的个人数据，只有在您点击“加载所有外部内容”后才会显示外部嵌入内容：*我同意显示外部内容。这将导致个人数据传输给第三方平台。详细信息请参阅我们的[隐私政策](/datenschutz/)*。

*据称，此技术支持多个重要特性，例如多核心负载平衡和实时加载资产的流媒体系统。如果该项目真正有效，将来在浏览器中展示图形更加复杂的游戏应该是可能的。据称，更多细节将会[很快公布](https://twitter.com/AlexStLouis10/status/1757909424805757239)。除了 Unreal Engine 5，该公司还计划未来支持其他引擎，如 Unity 和 Open 3D Engine。*

***值得一提的是:** [**使用 Unreal Engine 5 制作 RPG：前《巫师》开发者需要更多时间**](https://www.pcgameshardware.de/The-Thaumaturge-Spiel-74216/News/Release-Verschiebung-1440796/)*

*将 AAA 游戏未来能够通过网页轻松游玩，无需在本地硬件上安装的梦想变为现实，可能仍然遥遥无期。毕竟，尽管如此，必须下载相应的文件，对于大型项目来说这几乎不太实用。此外，WebGPU-API 的支持目前仍然非常有限。根据[Can I Use](https://caniuse.com/?search=webgpu)的说法，只有最新版本的 Chrome、Edge 和 Opera 支持它。因此，Web浏览器游戏暂时仍然主要依赖于 WebGL。对于这种API，也有一些替代引擎。例如，自由的游戏引擎Godot也支持相应的Web导出 - 目前正在努力支持WebGPU的实现[这里正在进行中](https://github.com/godotengine/godot-proposals/issues/6646)。*
