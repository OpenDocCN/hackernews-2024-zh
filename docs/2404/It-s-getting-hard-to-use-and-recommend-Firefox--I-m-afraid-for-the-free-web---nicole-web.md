# 使用和推荐Firefox变得越来越困难了，我对自由网络感到担忧

**2024年4月8日，星期一**

几个月前，我个人电脑上的每次视频通话最终都使用了Chromium。我尝试使用Firefox，在我的端看起来很好：我能看到和听到对方。但他们只看到空白的视频 feed，并听不到任何声音。Firefox显示我正在发送，但它永远不会传递给他们。这发生在Google Meet、一个身体倍增平台和一个远程医疗平台上，它们都使用不同的视频服务。

我是一名软件工程师，在我的个人笔记本上运行Fedora。这[特定的Bug](https://bugzilla.mozilla.org/show_bug.cgi?id=1875201)出现在最新版本上，由于我运行的是最新版本的分发版，我遭到了影响。在我的工作笔记本上，运行Ubuntu的LTS版本，我并没有遇到同样的问题。因此，在某些方面，这是我自己造成的问题，而且有[一种解决方法](https://discussion.fedoraproject.org/t/firefox-122-workaround-to-fix-google-meet-issue/103667)。

但问题是这样的。我必须对我安装的Firefox版本非常*小心*，因为有时会发生严重的问题。即使在最前沿，像这样的致命错误Bug——这不是我第一次遇到类似的情况——意味着我*不能*仅依赖Firefox作为我的唯一浏览器。我必须在Chromium中也设置好一切，因为Firefox*一定会*让我失望。

从某些方面来看，持续的故障并不是Firefox的错。当Web应用程序仅在Chromium中进行测试时，它们将不可避免地在非Chrome浏览器中出现显示器致命错误的错误。然而，其他问题似乎确实是Firefox的问题，比如这个视频通话的问题。但归根结底，“责任在谁”的问题并不重要。如果用户不能可靠地使用浏览器进行日常任务，他们将转向其他浏览器并永远不再回头。无论他们是转向Chrome、Edge还是Brave，这些浏览器实质上都是Chrome引擎。

使用Firefox变得越来越困难了，但我还在使用，因为保护自由网络需要我们除了Google（和苹果）之外还有更多的利益相关者。过去，我们在维护自由网络时有Google、苹果、微软和Mozilla都在桌子边。但现在微软使用Chromium引擎，虽然苹果通过iOS具有强大的影响力，但由于欧洲监管机构要求他们在iOS上允许替代的浏览器引擎，这种影响力可能会减弱。

目前几乎不可能为其他人使用 Firefox 辩护。自由网络的抽象损失，将控制权交给 Google，这些问题虽然无法触及，但与每天使用较差的浏览器带来的真实痛苦相比，其影响微不足道。我们目前对浏览器引擎多样性的好处过于远离，难以提出有力的论据。

希望在 Mozilla 的新 CEO 领导下，他们能够重新站稳脚跟。他们正在重视数据隐私。并非每个人都关心这一点，但关注的人足够多，也许我们将重新找回 Firefox 对普通日常用户的明确定位。目前，我仍然默认使用 Firefox，但每次遇到重大错误都让人犹豫不决。也许，总有一天，我会改变不再首选它了。

我害怕那一天的到来。
