# TBM 271: 最大的未开发机会 - 由约翰·卡特勒（John Cutler）撰写

> 来源：[https://cutlefish.substack.com/p/tbm-271-the-biggest-untapped-opportunity](https://cutlefish.substack.com/p/tbm-271-the-biggest-untapped-opportunity)

浪费最大的来源是*不是*低绩效者或者员工过多。

浪费最大的来源是未开发的**熟练的实用主义者**。

任何公司中熟练的实用主义者的数量比人们意识到的要多得多，因此代表了一个巨大的机会。但几乎可以说，他们的能见度较低。

熟练的实用主义者太聪明了，不愿陷入冲突或政治操控中。他们在系统内工作。他们*被低挑战*和*无聊*但足够机智以其他方式满足自己。他们不懒惰——只做最低限度然后停止——但他们也不会“超越期望”，因为他们本能地怀疑那是否值得。他们非常可靠，但只到他们方便（对他们来说）隐形的程度。偶尔，他们展现出闪光的才华提醒别人他们的能力，但不是经常，以免自己置身于狮子口中。

重要的是，这些员工抵制管理层“激励和挑战”的努力。他们不需要被激励——他们相对平衡。如果在现有系统内工作不那么困难的话，他们会乐于接受挑战。“我在工作外有我的项目来挑战我自己；非常感谢。”因此，正常的策略并不奏效。那感觉傲慢和家长式管理。

*(注：在继续之前，我想指出，熟练的实用主义者通常有特权，可以在某种程度上保持超然，但仍能谋生。许多人没有享受到这种隐身的特权，他们更多地接受审视、压力、精神虐待和怀疑。)*

让我们探索一个典型熟练的实用主义者的例子，让这一切变得真实。

玛丽亚非常清楚她的团队在原地打转。她略带娱乐地看着管理者徒劳地推迟对反应性功能请求的抗争。她技术娴熟，能在团队继承的乱七八糟的代码库中取得不错的进展，但她知道自己能更高效率。

在站立会议中，玛丽亚没有浪费精力去指责那些超出她（和团队）直接控制范围的事物。是的，软件工程是她的激情所在，但她不打算通过搅局来消耗大量社会资本——她看到了*那些*人的下场。她有一些同事非常强烈的事业主义倾向。在某些情况下，他们利用绩效管理系统到了极致，却损害了客户的成果。她知道升职需要更长时间，但她也有运气不错的时候，每隔几年就换公司升职加薪。几乎像是新的招聘经理察觉到她一直被低估。

Maria是一个技术实用主义者。她不是软弱或表现不佳者。她是一个在我们称为“公司”的怪异社会技术系统中航行的人类。

然而，问题在于这里。与其想出如何利用Maria不可思议的潜力，公司：

+   听到房间里的响亮声音，有关于什么是破碎的强烈意见。你错过了关于如何修复问题的整个信息来源。

+   专注于明显的“低表现者”。这很有趣，因为你可能有大约5%的员工是低表现者。然而，有60-80%的Maria。

+   专注于奖励10倍英雄主义和竞争激烈的职业生涯主义者

+   错误地将问题视为“参与”问题。

+   责怪一线经理未能在问题存在时与Maria进行互动。Maria变成了“管理问题”而不是领导问题。

+   减少Maria的贡献，因为它们不那么可见和炫耀。

+   提升那些积极戳和敲打系统的人，或者那些在政治上足够精明以应对系统的人——这会导致维持任何存在的功能障碍。当掌权者在导航事物方面很擅长时，为什么要改变事物呢？

当公司坐拥人才、动力和想要挑战但未受到挑战的人时，他们正在裁员！

如果你想要这里的第二部分来解决这个问题。这里是一些暂时的结论思考。

我在想我最初是因为什么而吸引我进入技术。好玩！作为一个问题/难题解决者和与其他同样好奇的人一起建立/制造可以影响人类的东西，被付费真是不可思议。那太不可思议了。有时我不得不掐自己一下。

我意识到我的一个大触发器是不一致、失调和浪费。许多公司都充满了喜欢解决问题的人才，然而，我们在利用他们的潜力方面只是皮毛。

我们让自己如此艰难。就像我们在 sabo自己。我不确定为什么——也许人类将是人类，也许是权力和金钱，也许是公司和产品的自然熵——但我们在工作中面临的许多问题是可以解决的。然而，出于某种原因，它必须是困难的。

或者呢？

而且随着那个……考虑一下你公司中未开发的Maria潜力。
