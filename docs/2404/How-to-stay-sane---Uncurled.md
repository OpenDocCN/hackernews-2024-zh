# 如何保持理智 | Uncurled

> 来源：[https://un.curl.dev/life/stay-sane](https://un.curl.dev/life/stay-sane)

我编写的代码几乎在每台连接到互联网的设备上都能运行。这种情况可能会给你的生活带来压力。

+   拥有数十亿用户的压力

+   报告的（许多）错误的压力

+   投诉/滥用用户的压力

所有这些都可能带来心理负担，逐渐消耗你的精力。

我处理这些潜在压力领域的方法包括：

测试案例

随着用户数量的增加，应该增加测试案例的数量。通过为新功能和新 bug 修复添加新的测试案例，随着时间的推移，测试越来越多，做另一个发布或更改内部结构的行为变得不那么危险。测试案例加上长时间内的多次发布，可以逐步建立起基础功能的信任水平。之后进行的发布会变得不那么令人紧张。增加另外十亿用户应该没问题。无需担心。

Bug 并不那么可怕

大多数 bug 可以解决，用户通常甚至可以在需要时降级到旧版本。即使用户在使用您的代码时遇到问题，也很少是您感到紧张的原因，即使该用户有一个已知的品牌。即使该用户似乎感到不满或行为侵略。

提出大量问题。弄清如何重现。分析。调试。修复。并保持冷静。代码是开源的，即使作为维护者，您的主要任务也是帮助用户调试问题。

挑衅用户

代码按原样提供，不带任何保证。即使是最喧嚣、最尖锐或使用最糟糕语言的人也知道这一点。不能遵守行为准则的用户应该立即被禁止和忽视。对于那些让人讨厌但没有越线的用户，我会努力在脑海中提取他们的技术论点，并且在不被引诱或其他恶劣行为误导的情况下进行讨论。另外，出于自己的心理健康考虑，我倾向于对那些难以共事的人进行“速率限制”；只要他们粗鲁，我会以越来越长的间隔回复。

此外，一个黄金法则当然是在感到沮丧或愤怒时永远不要真正发送您的回复或评论。最好是等待，睡一觉，第二天回来将其编辑成最基本的内容。

真实生活是一个很好的逃避方式

人们经常问我如何从现实生活中抽出时间来投入到开源项目中。尽管这本身就是一个挑战，但对我来说，“现实生活”意味着拥有一个有妻子和孩子以及其他业余活动的家庭，这也为我提供了一个健康的“对立面”，与我的开源活动形成了平衡。

投入所有时间、所有日子在开源项目上最终让我失去兴趣和精力。我需要分散注意力和生活中的变化，因此与家人共度时间并不是浪费开源项目的时间，相反，这是充电和必要的休息，让我能够带着更多的精力和更好的想法回来。

我不认为我需要告诉你们所有人，在与公婆吃晚餐时，即使远离键盘，你们仍然可以提出出色的想法并解决最棘手的问题。
