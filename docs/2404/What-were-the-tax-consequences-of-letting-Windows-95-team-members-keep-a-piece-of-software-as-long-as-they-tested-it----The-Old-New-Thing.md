# 让Windows 95团队成员测试软件并允许他们保留一份软件，这会导致税务后果是什么？ - 《新旧事》博客

# 让Windows 95团队成员保留软件作为测试的一部分会有什么税务后果？

偶尔会有人重新发现我关于[购买整个Egghead Software商店的故事](https://devblogs.microsoft.com/oldnewthing/20050824-11/?p=34463)。其中一个问题是税收问题。（大多数美国人今天都要交税。）

如果员工们被允许保留软件，前提是他们测试了程序并提交了错误报告，那么这是否算作工作补偿，从而成为应纳税收入？

事实上，没有人核实过你是否测试了你拿走的程序，对于参与活动的每个人来说都是显而易见的，因为你只是拿起软件走出房间，没有填写任何文件。你被认为是团队的好成员，不会出现没有打算做的活动。测试程序的期望只是期望而已，不是条件。

我的理解是，这使得软件被视为美国税法*de minimis*规则下的一份礼物。这是一次性的事情，因此满足频率标准。软件的价值远低于100美元，因此也满足了这一要求。而且由于没有人实际跟踪谁拿了哪个软件，因此也满足了行政不切实际的要求。

但实际上，这只是一个开发经理自行承担的一些非常规应用兼容性测试，并且同时培养团队合作精神的行动。

**额外的闲聊**：在这个故事的多次重述中，有人提到他们通过这个活动得到了一份《星际争霸III》的拷贝。我立刻记起，他们通过对Windows 95提交错误报告的方式达到了他们的期望：当你在58级或者什么地方获得了隐形装置时，你无法激活它。

幸运的是，他们在58级提供了一个存档，所以我不必自己玩到58级。

问题在于激活隐形装置的热键是`Ctrl`+`C`，这与在MS-DOS会话中取消活动剪贴板粘贴操作的`Ctrl`+`C`冲突。通常情况下，Windows 95会意识到没有活动的粘贴操作，然后重新播放热键到MS-DOS会话中。然而，热键的重播显然对这款游戏来说太快了，所以它从未激活隐形装置。

我通过改变 Windows 95 来修复了这个问题，使得它只在剪贴板粘贴操作活跃时安装`Ctrl`+`C`热键处理程序，并在粘贴操作完成后将其移除。这样，当你在粘贴操作之外按下它时，按键会以人类速度显示给游戏，从而使其能够启动隐身装置。
