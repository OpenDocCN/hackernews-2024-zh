# 微软必须停止将安全性作为高端服务销售 | Directions on Microsoft

> 来源：[https://www.directionsonmicrosoft.com/members/blog/2024-04-23/microsoft-must-stop-selling-security-premium-offering](https://www.directionsonmicrosoft.com/members/blog/2024-04-23/microsoft-must-stop-selling-security-premium-offering)

在一个完美的世界中，微软会再次认真对待安全性。它将对违规行为透明化。其高管将停止在微软自身员工甚至客户无法抵御的频率日益增加的事件发生时夸耀其增加的安全服务收入。微软会将必备的安全功能作为现有订阅的一部分，而不是将其作为附加组件出售。

将安全性作为高端服务来销售，帮助微软将安全性构建成一个自称年收入 200 亿美元的业务（截至 2023 年 1 月）。将 Microsoft 365 E5 作为获得核心安全工具的先决条件是微软安全增长战略的一部分。另一部分是将某些安全工具仅作为 Microsoft 365 E5 订阅的附加组件提供。

根据 *Directions on Microsoft* 分析师 Wes Miller 的说法：“客户需要工具来确保其组织的安全，而不必每年为微软的最新顶级安全服务付出越来越多的费用——这些服务越来越向更高的架子移动。”

### 最简单但最昂贵的选择：全部采用 Microsoft 365 E5

在微软让步之前，如果它曾经，将更多核心安全组件纳入基础 Microsoft 365 订阅，客户应该怎么做？

一些组织选择对所有知识工作者授权 Microsoft 365 E5，每用户每月费用为 57 美元，因为 E5 包含 E3 的所有功能，并添加了大多数（但不是全部）微软的顶级安全和合规服务。仅部分授权组织使用 E5 存在风险，因为安全和合规工具通常在整个 Microsoft 365 租户范围内启用，并且很少在每个用户级别执行许可证强制执行。

其他客户试图通过拼凑一堆单独购买的安全和合规组件，并将它们添加到价格较低的 Microsoft 365 E3 或 Office 365 E3 订阅中来绕开完全采用 E5。

这种方法可能对某些组织在财务上有意义，但现在不合算。多年来，微软提供了两个附加组件，Microsoft 365 E5 安全性和 Microsoft 365 E5 合规性（每个 12 美元），作为一种方法，使组织可以将 E3 用户带入许可合规性，而无需支付完整的 E5 价格。但由于2022年对几乎所有 Office 365 和 Microsoft 365 套件的价格上涨，现在通过这两个传统附加组件为用户授权安全性和合规性功能比直接授权 Microsoft 365 E5 更昂贵，而 E5 的价格已经有一段时间没有变化了。

### 微软迈出了一小步

微软可能会将核心安全功能纳入更多订阅计划，这并非完全不可能。在几次广为人知的攻击后，微软在这方面做出了一些让步。

OAuth，一种用于授权应用程序的协议，是Midnight Blizzard在2023年攻击微软时使用的关键组成部分。虽然基于OAuth的应用程序越来越普遍，并且现在越来越被利用来入侵组织，微软提供的工具来帮助监督却有限，并要求使用任何这些工具的Microsoft 365 E5。

风暴-0558邮件入侵去年袭击了微软及其一些重要政府客户，微软因其一些安全推销做法而面临投诉。可以检测到这一事件的日志信息仅供购买了高级E5计划的Microsoft 365客户使用。而使用E3的客户无法查看所需的日志信息。

微软官员承诺将更广泛地提供日志记录，并于2023年10月，将Purview Audit（标准版）的审计日志增加到180天，比最初的90天增加了一倍。对于通常使用Microsoft 365 E3（或更低版本）许可的客户来说，这意味着一旦日志配置最佳化，客户有180天的时间分析日志事件中的恶意行为，然后这些事件将完全丢失。但如果客户希望延长日志保留期限，他们仍然需要添加Purview Audit（高级版）以获取一年的保留期，以及10年的Audit Log Retention附加包，以获取长达10年的保留期。

微软是否会进一步妥协，使安全成为其更多云订阅的核心，以减少攻击的影响，更不用说坏的公众关系了？我们将拭目以待……
