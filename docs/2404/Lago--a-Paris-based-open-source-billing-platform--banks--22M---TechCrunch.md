# [拉戈](https://techcrunch.com/2024/03/14/lago-a-paris-based-open-source-billing-platform-banks-22m/)，一个总部位于巴黎的开源计费平台，获得了2200万美元的投资 | TechCrunch

> 来源：[https://techcrunch.com/2024/03/14/lago-a-paris-based-open-source-billing-platform-banks-22m/](https://techcrunch.com/2024/03/14/lago-a-paris-based-open-source-billing-platform-banks-22m/)

这家起初致力于构建营销工具的巴黎初创公司，在成功转型为计费平台后筹集了2200万美元，这个领域在潜在客户中明显存在问题。[拉戈](https://www.getlago.com)，开发一款开源计费平台的公司，已经在两轮融资中筹集了资金，与其官方发布同时宣布。

今天正式推出，但在封闭测试期间已经运营了一段时间，在这段时间里，它吸引了一些知名初创企业，如Mistral.ai、Together.ai和Juni，作为早期客户。公司CEO Anh-Tho Chuong表示，公司专注于开源是非常有意义的：它针对的是那些寻找解决方案以适应他们正在开发的最新创新服务的开发人员，这是市场上现有企业没有很好解决的空白，而拉戈打算通过开源方法来最好地解决这一问题。

“我们是开发者的伙伴，”她在接受采访时说。“我们尊重他们的抽象概念，并使用数据来计量使用情况，帮助公司轻松处理订阅或其他定价计划。”

一些知名投资者已经注意到：最新的1500万美元的A轮融资由FirstMark领投；之前的700万美元的种子轮融资由SignalFire领投，Chuong说。其他支持者包括Y Combinator、New Wave和Script，以及一些个人投资者，他们的参与强调了拉戈瞄准的市场细分。其中包括MongoDB的商业化负责人Meghan Gill；Romain Huet（曾在Stripe工作，现在在OpenAI担任开发者关系）；以及Hugging Face的CEO Clément Delangue。

我们从消息来源了解到，其估值现在约为1亿美元。

今天的拉戈作为一个计费平台开始经营，这完全是一个典型的创业公司方式：当初并不知道自己会成为一个计费平台。

当她和联合创始人拉菲·萨尔基西安在商业银行初创企业Qonto工作时，他们决定冒险自立门户，创建一个新的创业公司。他们申请了Y Combinator，并凭借他们的背景被录取进入了2021年夏季的批次。“但是我们去Y Combinator的时候还没有产品，”她说。

他们在市场营销方面取得了突破，特别是围绕构建“营销团队的Zapier”的理念。

“我们当时真的以为这会是一个大买卖，”她回忆道。“结果还好。” 然而，‘还好’显然不够。市场技术非常拥挤，公司几乎没有为其产品获得多少关注。

在试图通过增长黑客方式吸引观众的时刻，萨尔基西安决定为 Hacker News 写一篇文章，他在其中抱怨了[开发者计费问题](https://news.ycombinator.com/item?id=31424450)。

它有一个引人注目的标题：“对工程师来说，计费系统是一场噩梦”，而且它是以一种你可能只有在真正从心底创作时才能捕捉到的自由写作方式编写的。这是因为它是萨尔基西安和莊在 Qonto 期间了解良好的东西，因为他们的时间都花在解决这个问题的产品上。

莊说，这真的不是重点。发布这篇文章的动机是选择他们了解的东西，这样他们就可以监控参与度，并可能利用这一点来转移关注到拉戈，“营销团队的 Zapier”。

但这篇文章触动了许多人，令他们惊讶的是，很多人开始谈论自己的计费问题。拉戈迎来了“顿悟”时刻：如果他们真正想做的是为开发者解决问题，这里有一个问题他们确实能解决，并且他们知道他们能做得很好。于是，他们转向计费，并且这个初创公司开始起飞。

不仅与用户一起起飞，还与投资者一起。

“我们最初是通过 HackerNews 在 2023 年初发现拉戈的：他们有如此多的动力，对一个看似已解决或琐碎的话题来说，人们明显一直在等待像他们这样的开源解决方案。然后我们联系了他们在 GitHub 上的 star 见证者，反馈都是极好的，”SignalFire 的合伙人奥安娜·奥尔特安（Oana Olteanu）在一份声明中说。

如果你认为营销是一个竞争激烈的市场，那么计费比星期五早上 Billingsgate 鱼市还要拥挤。像 Stripe、Adyen、Salesforce、Zoho、Paddle 等大型科技公司提供计费解决方案。甚至已经有许多供应商在追求开源方法，包括 FOSSBilling、ChargeBee、Kill Bill、AppDirect 的 jBilling 和富有想象力命名的“开源计费”。（为什么绕来绕去呢？）

莊认为，依然有很大机会专注于可扩展性，为试图在自己竞争领域推动边界的初创公司提供量身定制的解决方案。

在她看来，人工智能领域就是一个很好的例子。那些构建基于人工智能产品的公司仍在摸索什么样的商业模式是可行的，与此同时，我们看到许多公司在考虑混合方法，将固定费率订阅与消费基础定价相结合的例子比比皆是。所有这些都很难管理，并依赖于能够与开发者构建的任何东西集成，并能够识别和应用他们的使用数据的工具。

“如果你有非常简单的定价和计费方式，市场上有很多这样的解决方案，但对于复杂的计费方式，一直没有很好的解决方案，”她说道。这导致很多公司（如Qonto）都在自行构建解决方案。“但工程师们讨厌这种方式。而且明显雇用工程师的成本也非常高。所以这仍然是一个未解决的问题。” 拉戈认为，提供开源工具是满足各种需求和想法的最佳解决方案。

对于一些用户来说，开源的理念也与他们希望作为企业所信奉的价值观相契合。

“我们选择拉戈作为我们的计费提供商，因为我们相信开源生态系统，” Mistral.ai的联合创始人兼CTO Timothée Lacroix在一份声明中说道。“他们能够跟上我们发布的步伐，并且让我们专注于我们擅长的领域。” 是的，一些人可能会认为，开源现在可能被视为一个概念被过度利用，很可能与许多所谓的开源公司正在构建的内容完全相反。

公司的目标是继续发展现有的业务，同时考虑未来可能增加的内容。一个明显的方向是重新探索拉戈最初关于营销的想法，并为客户提供更多关于他们所消费和支付的数据分析，以及他们支付模式的信息。另一个方向是探索计费的另一面：支付。

拉戈不太可能构建一个支付堆栈；然而，她补充道：重点几乎肯定会放在支付编排上，让用户可以控制他们使用的内容，但要确保它能很好地与他们的计费平台集成（理想情况下，当然是由拉戈提供支持的）。
