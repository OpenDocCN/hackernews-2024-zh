# Nadim Kobeissi：我们需要谈谈桌面上日历软件的现状

> 来源：[https://nadim.computer/posts/2024-04-18-calendar.html](https://nadim.computer/posts/2024-04-18-calendar.html)

## 我们需要谈谈桌面上日历软件的现状

2024-04-18

智能手机没问题。如今任何智能手机都可以找到好的日历软件。但是当谈到台式机（或笔记本电脑）时，仅有两种情况下，使用2024年的日历软件不会完全失败：

1.  你正在使用你的日历提供商的Web应用程序，

1.  你在使用Mac。

其他情况都会导致你陷入完全过时、未维护和不可用的日历软件的泥潭中：如果你使用Linux或Windows，**尤其是**如果你想要链接来自不同账户（如Google、Outlook、CalDav等）的多个不同日历。

这简直太疯狂了。日历很重要。我们每天都在使用它们。它们怎么会在桌面上如此失灵，*这可是我们完成工作的地方*，除了两种特定情况？不，我很抱歉，Thunderbird的日历不是一个好的日历。它是1998年的对话框，发出那种*“咚！”*声音，不会立即被关闭，而且还会无故弹出两次，或者告诉你修改了事件，其实你并没有。这不是我谈论“好的日历软件”时的想法。

### 我们怎么会走到这一步？

很简单：这是一个极不时尚的话题*和*一件没有人感激的工作。谁愿意写日历软件？谁会为日历软件买单？！再加上Web应用程序对大多数人来说通常足够了，因为他们通常只需担心一个日历，通常是他们的工作日历或个人日历，两者都由Google、Outlook或iCloud提供。

自愿修复日历软件让我想起[Tal Rasha自愿](https://youtu.be/MKNoy3m5X5Q?si=H7JKYwPSWvU3BIvk&t=95)锁住鲁特戈林中的封印符文，以永久将巴尔的灵魂石绑定在他体内。谁会想把那当成自己的命运？用你的时间做更好、报酬更高、不那么繁重而且更有声望的事情，远比这好得多。这几乎和写邮件客户端一样糟糕。我甚至没有勇气写一篇*博客文章*来谈论需要更好的邮件客户端的问题。

但是*如果你有多个来自不同账户的日历需要同时查看*，就像大多数技术专业人员一样，*如果你只是想要一个该死的桌面应用程序*，那么，在这种情况下，你唯一的幸运之处就是成为…

### 独特的Mac例外情况

出于某种原因，macOS 拥有一个非常竞争激烈的桌面日历软件生态系统。除了苹果内置的日历应用外，还有[BusyCal](https://www.busymac.com/busycal/)和[Fantastical](https://flexibits.com/fantastical)，只是举几例。它们都以天价售卖并销售大量副本！BusyCal 售价$49.99，而Fantastical只提供订阅方式。尤其是BusyCal，简直疯了。你能想到的每个功能，每种日历协议。这是世界一流的日历软件。真正的日历豪华车。

但我们没有多平台的或开源的替代品。那么，你知道的，大多数电脑怎么办？我们怎么能忍受**大多数工作站除了网页浏览器外根本没有可用的该死日历**？

### 必须采取措施

我的看法是，摆脱这种大多数工作站没有良好日历的荒谬泥潭的唯一现实可行解决方案，可能就是我们得撸起袖子，着手创建一个新的开源日历项目。最有前途的方法几乎肯定是利用[Tauri](https://tauri.app)。为什么选择Tauri呢？

+   Tauri 整合了 Rust 和 TypeScript —— Rust 提供性能和安全性，TypeScript 提供开发和维护的便利性。两者之间有一个桥梁。

+   Tauri 擅长创建具有极低开销的多平台应用程序。这对于现有 Electron 应用程序的核心痛点——臃肿和低效率表现——至关重要。

通过在 Tauri 上构建，我们可以针对多个平台，并保持统一的代码库，这简化了更新和修复错误。

所以，在我对桌面日历软件凄惨状态的怒吼结束时，你站在这里。我们可以每天早上继续抱怨，或者我们可以真正做点什么来解决这场闹剧。说真的，我们还能容忍多久这种荒谬？蓝图已经在那里：基于 Tauri 构建的时髦多平台日历应用程序，等待从我们集体的激愤和天才之火中锻造而出。如果这一切都不能激励你采取行动，那或许你可以继续与你所遭受的过时工具进行斯斯菲斯的奋斗。

或者，如果仁慈的精神（或者沮丧）足够感动您，就向我提供一笔资助吧——我将承担起编写这个神秘软件的艰巨任务。我将成为你的塔尔·拉莎！
