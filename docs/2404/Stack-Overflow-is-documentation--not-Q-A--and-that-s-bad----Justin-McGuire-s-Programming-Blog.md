# Stack Overflow 是文档，不是问答，这很糟糕 :: Justin McGuire's 编程博客

> 来源：[https://landedstar.com/blog/posts/stack-overflow-is-documentation/](https://landedstar.com/blog/posts/stack-overflow-is-documentation/)

旧的 Perl 新闻组 comp.lang.perl.misc 部分用于讨论，但也用于回答问题，对初学者非常不友好。它有一个庞大的 FAQ，老手们经常对那些已经有答案的问题大声叫嚷。可以理解，因为没有人喜欢重复自己。但初学者是最难理解整个过程的人，也是最需要帮助的人。新闻组设定了他们自己，不帮助那些最需要帮助的人。

(顺便说一句，php.net 网站非常有帮助。每个函数下都有一个扁平的讨论区，按最近的排列，天哪，这几乎回答了我大部分的问题。)

然后 Stack Overflow 出现了，所有人都搬到那里去了。那些无用的“阅读 FAQ。[plonk](http://www.catb.org/jargon/html/P/plonk.html)”评论在踩声中消失了，而有帮助的评论则被赞了起来。

在某个时候，那种“有帮助”的氛围发生了变化。Stack Overflow 不再是一个问答网站，而开始变成一个文档网站。问题被回答为“看这个其他答案”。与新闻组相比，这里的粗鲁少了，但接近了同样的不友好。

或许所有问答网站最终注定会变成文档网站的命运。如果你一次“正确”地回答了问题，为什么还要重复呢？

但我认为重复回答是有益的。它让下一代程序员练习他们清晰表达和回答问题的能力。它让他们收集他们的赞和徽章。有一个健康的漏斗让初学者学会如何提问，老手学会如何回答问题，而超级老手学会如何进行管理。
