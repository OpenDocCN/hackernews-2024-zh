# [美国政府面临的微软问题](https://www.wired.com/story/the-us-government-has-a-microsoft-problem/) | WIRED

> 来源：[https://www.wired.com/story/the-us-government-has-a-microsoft-problem/](https://www.wired.com/story/the-us-government-has-a-microsoft-problem/)

当[安全专家](https://www.sentinelone.com/blog/the-microsoft-paradox-dominance-vulnerability-in-the-world-of-cybersecurity/)越来越批评[微软未能](https://www.cybereason.com/blog/microsofts-failure-to-prioritize-security-puts-everyone-at-risk)及时和[充分修复](https://arstechnica.com/security/2024/03/hackers-exploited-windows-0-day-for-6-months-after-microsoft-knew-of-it/)其产品中的[漏洞](https://therecord.media/microsoft-resolves-vulnerability-following-criticism)，这些事件发生了。作为美国政府迄今为止最大的技术提供商，微软的漏洞占据了[绝大多数](https://www.sdxcentral.com/articles/news/microsoft-vulnerabilities-make-up-75-of-top-20-exploited-list/2023/09/)新发现的以及[最广泛使用的](https://www.cisa.gov/news-events/cybersecurity-advisories/aa23-215a)软件缺陷。许多专家称，微软拒绝进行必要的网络安全改进以应对不断变化的挑战。

“微软没有根据威胁调整他们的安全投资水平和思维方式，”一位知名的网络政策专家说。“这是一个巨大的失误，而微软却有足够的资源和内部工程能力。”

国土安全局的CSRB在[其新报告](https://www.cisa.gov/sites/default/files/2024-04/CSRB_Review_of_the_Summer_2023_MEO_Intrusion_Final_508c.pdf)中支持这一观点，称微软展示了“一种将企业安全投资和严格的风险管理置于次要位置的企业文化”。该报告还批评微软在[最新中国入侵可能原因](https://msrc.microsoft.com/blog/2023/09/results-of-major-technical-investigations-for-storm-0558-key-acquisition/)的技术调查结果中发布了不准确的信息。

最近的数据泄露显示，根据多位专家的说法，微软未能实施基本的安全防御措施。

安全公司CrowdStrike的情报高级副总裁亚当·迈尔斯指出，俄罗斯人能够从测试环境跳转到生产环境，“这种情况不应该发生”，他说。另一位在微软竞争对手工作的网络安全专家指出，中国能够通过一个入侵窃听多个机构的通信，与CSRB的报告一致，该报告批评微软的认证系统允许单一签入密钥获取广泛访问。

“你不会听到其他云服务提供商出现这类事件的消息，”Meyers说。

根据CSRB报告，微软“没有充分优先考虑重构其传统基础设施，以应对当前的威胁形势”。

微软在回应书面问题时告诉《WIRED》，它正在积极改善其安全性以应对最近发生的事件。

“我们致力于适应不断变化的威胁形势，并与行业和政府进行合作，共同防范这些日益复杂的全球威胁，”微软联邦安全业务首席技术官Steve Faehl表示。

作为其于11月推出的[Secure Future Initiative](https://www.wired.com/story/microsoft-secure-future-initiative/)的一部分，Faehl表示，微软已经改善了其自动检测和阻止员工账户滥用行为的能力，开始在网络流量中扫描更多类型的敏感信息，减少了单个身份验证密钥所授予的访问权限，并为希望创建公司账户的员工制定了新的授权要求。

微软还重新部署了“数千名工程师”来改进其产品，并已经开始至少每周两次召开高级管理人员的情况更新会议，Faehl说。

这一新举措代表着微软“回应了CSRB报告中所指出的大部分优先事项和承诺”，Faehl说道。但是，微软并不认同CSRB报告所声称的其安全文化有问题。“我们非常不认同这一描述，”Faehl说，“尽管我们承认我们并不完美，还有很多工作要做。”

## 安全收入的“依赖”

微软因为[向其客户额外收费](https://www.microsoft.com/en-us/microsoft-365/enterprise/microsoft365-plans-and-pricing)，以提供更好的安全保护，如威胁监控、防病毒和用户访问管理，而在网络安全社区中招致特别的敌意。2023年1月，[该公司宣布](https://www.microsoft.com/en-us/security/blog/2023/01/25/microsoft-security-reaches-another-milestone-comprehensive-customer-centric-solutions-drive-results/)其安全部门的年收入达到200亿美元。

“微软已经将其安全性视为一种为他们带来收入的方式”，安全公司SentinelOne的研究副总裁Juan Andrés Guerrero-Saade说道。他的同事Alex Stamos [最近写道](https://www.sentinelone.com/blog/microsofts-dangerous-addiction-to-security-revenue/)，微软对[这种收入](https://www.geekwire.com/2022/microsoft-aims-to-boost-growing-cybersecurity-business-with-new-managed-services/)的“依赖”“严重扭曲了他们的产品设计决策”。
