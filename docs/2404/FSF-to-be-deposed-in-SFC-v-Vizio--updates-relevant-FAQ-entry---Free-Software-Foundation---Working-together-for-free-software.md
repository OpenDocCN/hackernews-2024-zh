# FSF to be deposed in SFC v Vizio, updates relevant FAQ entry — Free Software Foundation — Working together for free software

> 来源：[https://www.fsf.org/news/fsf-to-be-deposed-in-sfc-v-vizio-updates-relevant-faq-entry](https://www.fsf.org/news/fsf-to-be-deposed-in-sfc-v-vizio-updates-relevant-faq-entry)

近25年来，自由软件基金会（FSF）的"[关于GNU许可证的常见问题](https://www.gnu.org/licenses/gpl-faq.html.en)"一直是关于GNU通用公共许可证（GPL）的重要教育资源。

遗憾的是，美国上市公司Vizio, Inc.，一家设计和销售电视、音响、观众数据和广告的公司，试图利用FSF的FAQ对软件自由造成损害，而这涉及[*Software Freedom Conservancy (SFC), Inc. vs. Vizio, Inc.*](https://sfconservancy.org/copyleft-compliance/vizio.html)（案件号30-2021-01226723-CU-BC-CJC）。在此案中，SFC要求Vizio提供某些由GNU通用公共许可证v2（GPL）和GNU宽松通用公共许可证v2.1（LGPL）覆盖的Vizio设备上的程序源代码。作为回应，Vizio辩称，如果没有持有基础软件版权，SFC无法强制要求提供源代码。为支持这一观点，Vizio参考了一个标题为["谁有权力执行GPL."](https://www.gnu.org/licenses/gpl-faq.html#WhoHasThePower)的FAQ。

**Vizio现在已经传唤FSF出庭作证。**

自始至终，FSF 都希望GNU许可证促进软件的[自由运行、复制、分发、学习、修改和改进](https://www.gnu.org/philosophy/free-sw.html)——这要求源代码与用户共享。

Vizio试图利用FSF的FAQ来避免在GNU GPL下承担责任，这既是恶意的，也是讽刺的。用户应自由行使他们在GNU GPL许可下获取源代码的权利，无需依赖版权持有人采取行动。

**为此，我们已经更新了上述[FAQ](https://www.gnu.org/licenses/gpl-faq.html#WhoHasThePower)如下所示：**

> Q: 谁有权力执行GPL？
> 
> A: 由于GPL是一个版权许可证，它可以由软件的版权持有者来执行。如果您发现GPL的违规行为，请通知涉及的GPL覆盖软件的开发者。他们要么是版权持有者，要么与版权持有者有联系。
> 
> 此外，我们鼓励用户利用任何合法机制获取完整对应的源代码，这是他们的权利，并且要求全面遵守GNU GPL。毕竟，我们制定GNU GPL是为了使软件对所有用户都自由。

此更新更好地反映了GNU GPL许可证的意图。

我们正在准备就 Vizio 的传票提供证词。我们将继续密切关注此案件，并支持 SFC 的工作。毫无疑问，法院确认用户有权根据 GNU GPL 许可证执行其源代码权利的裁决，将对确保软件自由大有帮助。

自由软件基金会（FSF）在维护 GNU GPL 许可证的执行方面投入了大量有限资源，包括与法律顾问的广泛准备工作。请考虑通过[财务捐赠](https://www.fsf.org/donate)来支持我们继续这项工作。如果您没有财力，也可以为[GNU 项目](https://www.gnu.org)做贡献，或帮助调查[可能的 GPL 违规行为](https://www.gnu.org/licenses/gpl-violation.html)。
