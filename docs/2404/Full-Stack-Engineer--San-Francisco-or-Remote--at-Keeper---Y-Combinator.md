统称为零工经济、激情经济或兼职，如今超过一半的Z世代工作者都是用1099合同获得报酬。税法很复杂，许多自由职业者感到不知所措。他们支付不起个人簿记员，并且没有时间深入挖掘什么算作抵扣。结果就是他们在纳税时要损失数千美元。

Keeper的使命是通过为每个人提供个人会计师来解决这个问题。Keeper了解税法的细微差别，能在几分钟内回复，还能根据用户上传的表单创建个性化的纳税体验。

## 你可能要解决的问题

+   **基于上下文的智能**：当用户向Keeper提问时，我们希望给出与用户所在屏幕相关的答案，并将用户链接到应用程序的不同操作和不同部分。

+   **收入跟踪**：我们应该能够监控和预测收入以及开支。这将帮助用户全面了解他们企业的会计方面，并为我们提供预测用户四月税单的关键信息。

+   **季度税金**：自由职业者应该根据他们当季的收入留出和缴纳税款。这是一项需要手动操作的任务，因州而异，有时需要邮寄实体表格。我们希望能让用户付费让我们代劳。

## Keeper的工程师

这是在Keeper工作的工程师类型。

+   **您想要很多自主权，并且能够独立生产出优秀的结果。**在Keeper，您拥有自己的路线图，拥有自己的功能端到端（从与产品经理头脑风暴到在App Store上发布），并且您对技术决定有相当的自主权。

+   你专注于产品和用户。我们使用React、Node和SQL是因为它们是标准。我希望我们面临的最大问题是如何决定构建什么以及关注哪些用户痛点，而不是解决技术问题。

+   **你喜欢帮助他人。** 你帮助工程师调试棘手的问题，与 PM 集思广益，帮助客户支持解决我们系统的问题。

了解这些价值观的一种方法是将它们视为一组对比。

+   **自主性：** 你需要对自己要求高标准，迫使自己快速行动，并努力成为一个有代码精湛技艺的工匠。如果你没有自我激励去做这些事情，Keeper 可能不适合你。

+   **产品关注：** 虽然我们留出时间用于新技术和重构，但这并不是主要关注点，主要由产品需求驱动。

+   **协作：** 虽然其他组织可能有专门的跨职能沟通角色（例如，PM 或技术负责人），在 Keeper，工程师直接与利益相关者合作。这要求工程师更负责地对待与利益相关者的响应。

## 职责

+   与 PM 协作，提出新特性和解决已发现问题的新方法
