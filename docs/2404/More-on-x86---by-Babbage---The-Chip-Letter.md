# 更多关于x86的信息 - 由Babbage撰写 - The Chip Letter

*一个快速的编程公告。这是本周的两篇短文之一。这是上周[The Paradox of x86](https://thechipletter.substack.com/p/the-paradox-of-x86)的后续内容。下一篇将探讨现代半导体发展的历史时刻。*

上周关于x86的文章时机非常合适，因为它与

写了关于x86指令集架构是否需要“消失”的文章。

Casey在与Twitcher [ThePrimegen](https://www.twitch.tv/theprimeagen)关于这个话题的讨论之后，为了记录永久性，在YouTube上进行了讨论。

整个讨论是由两篇最近的文章引起的。第一篇是Hackaday的[为什么x86需要消失](https://hackaday.com/2024/03/21/why-x86-needs-to-die/)，这反过来又促使了Chips and Cheese博客的[为什么x86不需要消失](https://chipsandcheese.com/2024/03/27/why-x86-doesnt-need-to-die/)的回应。

Casey站在“不需要消失”的一边，并提出了一系列非常有力的观点，几乎所有观点我都同意。他总结了他的立场如下：

> 对于这个观点，我不同意x86“需要消失”的说法。我这么说的原因是因为x86（现在是x64）在过去四十年中在稳定软件开发方面做得非常出色。当然，你可以称之为“遗留问题”或者“维护问题”，如果你想看到它的负面影响的话。但是如果你愿意看到它的优势，你也可以称其为*非常灵活的指令集*。

Casey提到了Intel关于“春季清理”x86架构的APX提议，在去年七月它们公布时，我们曾经简要讨论过。

APX的关键提议包括：

+   将通用寄存器的数量从16个增加到32个；

+   新的三操作数指令（例如，增加了从寄存器2中减去寄存器1，并将结果放入寄存器3的能力）；

+   新的指令以一次PUSH / POP两个通用寄存器；

+   新的条件加载、存储和比较指令；

+   增加了抑制常用指令状态标志写入的选项；

+   新的64位绝对跳转指令。

Casey特别强调了抑制标志写入作为APX潜在性能优势的能力。

尽管我在很大程度上同意Casey的观点，但我认为这些讨论忽略了x86的其他方面。特别是，对于竞争和创新而言，与两家硬件供应商的绑定一直是一个严重的问题。在APX的帖子中，我们提出了这样一个问题：

> 为什么Intel / AMD在十年前没有这样做呢？

x86-64，即x86的64位版本，由AMD在1999年宣布。首批英特尔x86-64核心于2004年出现。第一批APX核心将何时出现？据我所知，英特尔尚未透露，这意味着最早要到2025年了。

我认为这些改进并非巧合，它们本应该在十年前就出现，但现在出现仅仅是因为x86面临了更多的竞争。如果我们需要一个展示x86“锁定”问题的示例，那么这就是它。

无论如何，我认为有些人称x86为“遗留”架构还为时过早。英特尔和AMD已经证明它可以保持竞争力，并且在APX的推动下将变得更加竞争力。正如我们之前讨论过的那样，x86将会长期存在 - 至少还有几十年，并且将继续在领先的制造工艺上大规模生产。它只是不再像过去几十年那样占主导地位，这无疑是健康的发展。

我意识到[《x86的悖论》](https://thechipletter.substack.com/p/the-paradox-of-x86)在某些有趣的细节上进行了简化。特别是AMD在x86开发中的角色以及“如果AMD也在进行x86设计，则它不是一个护城河”的潜在挑战。

英特尔和AMD在x86 CPU市场上已经是激烈的竞争对手将近五十年了。如果你想了解一切是如何开始的，那么

在下面的视频中，他像往常一样出色地完成了工作。

英特尔为与AMD竞争而做出的努力有时会溢出成为被判定为非法和反竞争行为。例如，欧盟委员会在2009年对英特尔的“裸露限制”行为罚款超过10亿欧元：

> ...英特尔实施的反竞争做法，损害了其在x86 CPU市场上的竞争对手AMD，在2002年10月至2007年12月期间。

（请注意，这项罚款后来被推翻，并在去年换成了更小的罚款。）

鉴于这种激烈的竞争和英特尔几十年来在制造领域的领先优势，AMD竟能够生存下来可能是非常值得注意的。

在写作本文时，AMD的市值超过了英特尔的（2360亿美元对1450亿美元）。这一成就得益于TSMC卓越的制造能力。即使在此之前，AMD也经常证明自己是一个灵活而创新的竞争者。当然，x86的64位版本是AMD的创造 - 称为AMD64 - 而英特尔则追求Itanium。

对于‘如果AMD 也在制造x86 CPU，这还能称之为壕沟吗’的挑战，壕沟只是一种防御障碍。在这种情况下，这是一种保护两家公司的障碍，只是不互相保护。

有人认为AMD 作为x86 CPU制造商的存在在几个方面对英特尔都是有利的。

+   延伸到物理壕沟的类比，当英特尔因其他事务分心时，AMD 填补了壕沟的偶发性漏洞，其中AMD64 是最显著的例子。

+   更近一步，随着英特尔在10纳米技术上的挫折，AMD 展示了x86 仍然是一种具有竞争力的架构，并可能延迟了一些向Arm 的转变。

+   或许更重要的是，我怀疑AMD 的竞争促使英特尔保持了‘比赛状态’，减少了可能导致公司更早衰退的自满情绪。

+   最后，我认为如果市场上没有AMD的存在，无论是在美国还是欧盟，对x86的反垄断关注都会更多。

我怀疑英特尔的管理层也认同这一点。也许是因此，AMD 的生存并不应该那么让人惊讶？！
