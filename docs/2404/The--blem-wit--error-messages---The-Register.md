# 'blem wit' 错误消息 • The Register

> 来源：[https://www.theregister.com/2008/01/28/error_messages/](https://www.theregister.com/2008/01/28/error_messages/)

年轻时，我收集了一堆系统错误消息。好吧，看，这没收集邮票那么让人难过！是吗？真的？哦好吧，不管了。无论如何，我的 [最近的文章](https://www.theregister.com/2007/12/19/quattro_pro_fcuk/) 让我开始重新审视那些收藏。

一些消息可以追溯到大型计算机的时代，比如：

"Fatal error in crawl out - system halted"

这条我从 Prime 收集到。

大多数错误消息在某种程度上得到了验证；然而，我曾经收集过一些令人愉快但未经验证的内容：“Shut her down, Clancy, she's a-pumping mud.” 无论其最初的来源如何，在六个月内，它已经成为现实——一个朋友在列表上看到它，并实际将其添加到了一个真实的系统中。据我所知，它至今仍在那里。这是艺术模仿生活的一个案例，或者反过来说…或者是科学，或者是其他什么。

然而，在那些日子里，大多数错误消息都没有那么有创意。原因？内存昂贵。因此，一些消息非常简洁。我的意思是 rlly trs - to th pnt of bng compltly uslss. 例如，可悲的：

"Cannot find DLL"

是的，是的，但是为了解决问题，我需要知道找不到哪个 DLL。这里有上百个该死的东西；至少给我一个提示。它以哪个字母开头？那会缩小范围。微软不明白为什么人们最终要对 Windows 大喊大叫。

但这还不算什么。我个人最喜欢的仍然是90年代初期的一条消息。它非常简洁，正如它们所能达到的那样：

"blem wit"

这些简单的字符证明了错误消息之神不仅有幽默感，还理解递归。

在某些版本的 Novell 重定向器 shell（NET5.COM）中存在一个 bug，那是一个在 DOS 下运行的 TSR（Terminate and Stay Resident）程序。Novell 的 TSR 当然有一组可用的错误消息。其中之一是：

"There is a problem with the Memory Control Block for the shell"

不幸的是，触发错误消息的问题非常不幸，还带来了一个副作用——使 DOS 无法正确写入屏幕。那么，到底是哪个错误消息呢？

"There is a pro**blem wit**h the Memory Control Block for the shell"

因此，每当出现 "blem wit" 时，你有两个理由确信确实存在屏幕写入问题。

"Blem wit" 是如此突然出现在屏幕上的奇怪字符串，并且在当时如此普遍，以至于经常被报告为病毒。但实际上它不是，它只是一个 TRS TSR。®
