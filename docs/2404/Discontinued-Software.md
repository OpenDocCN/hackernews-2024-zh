# Discontinued Software

> 来源：[https://almy.us/softwaredeath.html](https://almy.us/softwaredeath.html)

# The Threat Of Discontinued Software

我使用计算机已经超过50年了，所以我见过很多计算机程序和计算机的兴衰。但是当长期使用的程序不复存在时，总是令人不安的。当然，存储在专有格式中的数据是最危险的问题。但对工作流程的影响也是一个问题。我认为，现在我已经退休了，年纪也大了，这变得特别恼人。所以这里是我个人遇到过和正在遇到的一些问题。

## Changing Computers

I've gone from IBM 360 (and punched cards) to DEC PDP-11 to TRS-80 to CP/M to MSDOS and Windows with a stint using OS/2 and now for the past 15 years the Apple Mac. Each of those changes has generally meant a complete change of application software and, more often than not, new file formats. I'm concerned about changing again. Now that Apple is going to ARM processors, that is another change which would be difficult because of my reliance on discontinued/abandoned software that only runs on Intel Macs. And, of course, this software could still fail with future OS versions. The successors to macOS Mojave, in 2019, no longer support 32-bit applications, and this is a problem.

## My Java Saga

我在上个世纪90年代末开始编写Java程序。我使用了当时最先进的IDE——IBM的Visual Age for Java，它在许多方面都是最先进的IDE。我仍然可以在Windows 2000虚拟机上运行它，但它锁定在一个旧版Java上。IBM放弃了这个产品（实际上是用Smalltalk编写的），转而使用Eclipse，但我从未喜欢过Eclipse。我当时大量使用Java Beans，在Visual Age中得到了很好的支持，但在Borland JBuilder中也有支持，所以我转向了Borland。但后来Borland也转向了Eclipse。现在我使用NetBeans，尽管名字中有Beans，但对Java Bean的支持很少。现在软件的费用从每年约100美元降到了免费，但质量和功能大幅下降，所以我不是一个快乐的使用者。幸运的是，NetBeans跨平台（我在Windows、macOS和Linux下都使用过），仍在开发和支持中，所以我不觉得自己会再次孤立。但是所有权已从Oracle转移到了Apache，这确实延迟了他们的发布计划。

## IBM

Nobody has ever been fired for buying IBM, but this doesn't extend to their home products. With Visual Age the product was originally low enough priced for non-corporate users but the successor was much more expensive. I was also "burned" by OS/2\. They also had a good website design program for home users that they abandoned. And they also abandoned a tape backup program for OS/2 before they abandoned OS/2\. Basically every IBM product I ever bought ended up being abandoned.

## Photography

当我第一次开始使用数码摄影时，我仍然是基于Windows的计算机。我使用ACDSee和Photoshop Elements。我将图像文件保存在文件夹层次结构中。当我转向Mac时，我购买了Photoshop并使用Bridge进行管理，仍然保持图像文件的文件夹层次结构。我仍然可以使用购买的非订阅版本的Photoshop，但它无法在macOS Sierra之外运行，并且无法处理来自我的尼康D810的原始图像。

然后我迷上了Aperture，现在已经被抛弃。我对Aperture的放弃感到不满，因为Photos无法替代它，而我也不喜欢Lightroom。我还使用Nik插件，它们已经免费并被出售给了DxO，再次商业化。尽管Aperture在macOS Sierra中仍然可以运行，但那个版本将是Nik插件的终结，因为它们是32位的，而最新的DxO 64位版本不支持Aperture。这让我处于一个等待的状态，软件可用性仅剩一年。我保存处理过的图像为大型JPEG文件，比我实际需要的要大，希望以后不必再用其他程序重新处理图像。我确实保留了所有原始的RAW图像，以防万一。幸运的是，即使是来自我的第一台数码单反相机尼康D70的图像，现在仍然可以读取。

结果，有第三方程序允许Aperture在Catalina下运行，还有最新的DxO Nik插件。然而，这一切来得太迟，因为我已经转向Adobe Lightroom。

## 写作

幸运的是，我写过并且关心的最古老的文档都被打印出来了，我还扫描并OCR了它们。如果我仍然保存ASCII文本文件，它们仍然可以读取，而我使用的第一个文字处理文件是nroff格式，现在仍然可行！但随着个人计算机的普及，我使用了一系列现在已经不再支持的文字处理软件 -- Superscripsit（TRS-80），Wordstar（CP/M）实际上还可以恢复，PC-Write也可以恢复，还有一些非常好的OS/2文字处理软件，但无法恢复，并且多年来使用了可以被LibreOffice读取的WordPerfect。

但问题出现在我转向Mac之后！在Mac上没有WordPerfect，而且我不喜欢Microsoft Word，因为在90年代有过很多糟糕的经历。所以我转向了Pages。由于最新版本已经被削减以匹配iOS版本，这一切都变得不稳定。直到最近，最新版本的Pages甚至无法打开最早的Pages文档。我仍然在使用Pages，但是我还添加了Nisus Writer Pro。Nisus Writer Pro的优点在于其本地保存格式为RTF，几乎可以在所有软件中打开（但不包括Pages！）。Nisus Writer Pro还完全支持OS X功能，而LibreOffice/OpenOffice.org/Microsoft Word则不是。但是如今很少有人谈论Nisus Writer，那么它的未来会怎样呢？

我曾是Circus Ponies Notebook的重度用户。该公司突然关闭了。我继续使用这个程序，但其专有文档格式存在风险。幸运的是，可以提取文档内容。Circus Ponies Notebook在macOS High Sierra上停止工作。我已将这类工作转移到Growly Notes，当我第一次使用它时它是商业软件，但现在免费。它的未来将会如何？

## 服务器

苹果一直在从其Server应用程序中删除功能，我多年来一直在使用这些功能，并且基本上在2018年的Mojave中将其停产。他们不想再从事服务器业务，无论是硬件还是软件，这让所有当前用户陷入困境。我目前只将我的Mac mini服务器用作文件服务器，成功地将它执行的所有其他服务迁移到了UniFi Security Gateway（一个为小型企业设计的路由器）。

## 未来

我应该对来自小公司的程序持怀疑态度，但迄今为止失败似乎主要来自大公司的产品（除了Circus Ponies）。显然，像Scrivener这样拥有专有数据格式的程序存在着最大的风险。自1980年代末以来，我一直在使用Epsilon程序编辑器，它是由一个人操作的。听起来很冒险，但因为它只编辑ASCII文本文件，我不用真的担心，尽管如果它不在了我肯定会感到失落。在Catalina发布后，大约花了一年时间才将Epsilon升级到64位。

我遇到的大部分问题可以追溯到苹果每年发布新的macOS版本，带来了有限的新功能，至少对我有用的功能有限，同时禁用了我需要的旧软件兼容性。至少要保持安全补丁（当前版本和前两个版本）和一般修复（当前版本和前一个版本）。在我写这篇文章时，Big Sur（2020年）已发布，Catalina（2019年）获得支持但没有新版本，而Mojave（2018年）是最旧的获取安全更新的版本。但我遇到了以下情况：

+   我用2020年的iMac替换了我的2014年iMac，以获得额外的硬件支持年限，因为我需要Intel处理器来运行Windows虚拟机。它运行着Catalina，因为我不需要Big Sur。

+   我的2012年MacBook Pro也被替换了，因为它无法运行支持的操作系统版本。我现在有一台2019年的MacBook Pro运行着Catalina，基本上会遇到与我的iMac相同的问题。

+   我的Mac mini服务器因为苹果在High Sierra中取消了Server的功能，而在Mojave中实质性地放弃了这一产品，所以现在运行着Sierra。因为它现在是一个内部系统，我不再担心补丁。

+   我有另外两台Mac mini运行着Yosemite（2015年），因为它们无法运行更新的系统。

+   Joan的iMac也被2019年型号替换了。它也运行着Catalina。

一年前，我同时运行了四个不同的操作系统，但至少现在只剩下三个了！虽然花了一年多，但终于有了所有我需要的32位应用程序的替代品。现在我需要为基于ARM的CPU做好准备。
