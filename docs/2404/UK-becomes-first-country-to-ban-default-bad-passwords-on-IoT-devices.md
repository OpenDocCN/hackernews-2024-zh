# 英国成为首个禁止物联网设备上默认差密码的国家

> 来源：[https://therecord.media/united-kingdom-bans-defalt-passwords-iot-devices](https://therecord.media/united-kingdom-bans-defalt-passwords-iot-devices)

七年前，一场网络攻击导致许多美国最受欢迎的网站无法访问。2016年10月21日的三个延长时段内，互联网用户无法访问他们的Twitter、CNN和Netflix等其他热门网站。

自然地，关于可能造成如此大规模中断的强大威胁行为者的猜测甚嚣尘上。但这一事件并非由敌对国家发起。结果显示，这是一个极其简单的分布式拒绝服务攻击，目标是提供域名系统（DNS）服务的公司Dyn，这是互联网通信结构的关键部分。

虽然攻击并不复杂，但规模很大。发送到Dyn服务器的流量量是由一个由互联网连接的消费者设备组成的僵尸网络生成的，这些设备从无线摄像头到WiFi路由器不一而足。这个名为[Mirai](https://therecord.media/tag/mirai)的僵尸网络以日本动漫命名，由三名年轻的美国公民开发，他们几乎还未成年就被逮捕。

尽管他们在一年多后[认罪](https://www.justice.gov/opa/pr/justice-department-announces-charges-and-guilty-pleas-three-computer-crime-cases-involving)，但他们的发明引发了更持久的问题——仅凭松散的物联网设备生产商安全实践，特别是普遍使用的默认用户名和密码，使Mirai僵尸网络能够自动感染并传播到约30万台设备，并可命令这些设备攻击互联网上的任何其他内容。

周一，英国成为全球首个禁止使用默认可猜测用户名和密码的物联网设备的国家。仍允许默认安装独特密码。

[2022年产品安全和电信基础设施法案](https://www.legislation.gov.uk/ukpga/2022/46/contents/enacted)（PSTI）引入了制造商的新最低安全标准，并要求这些公司向消费者公开他们的产品将获得安全更新的时间。

制造和设计实践意味着许多物联网产品为它们连接的家庭和企业网络引入了额外的风险。在[一个经常引用的案例](https://money.cnn.com/2017/07/19/technology/fish-tank-hack-darktrace/index.html)，由网络安全公司Darktrace描述，黑客据称通过一个连接在鱼缸中的温度传感器进入了一家赌场的受到良好保护的计算机网络并窃取了数据。

根据PSTI，弱密码或易猜测的默认密码如“admin”或“12345”明确禁止使用，并要求制造商发布联系方式，以便用户报告漏洞。

不符合规定的产品可能会面临被召回，责任公司最高可面临1000万英镑（1253万美元）或其全球收入的4%，以较高者为准的罚款。

该法律将由产品安全与标准办公室（OPSS）管理，该机构隶属于商业和贸易部，而非独立机构。

消费者权益组织Which?的政策与倡导主任Rocio Concha表示：“OPSS必须为行业提供明确的指导，并准备对制造商采取严厉的执法行动，如果他们违法，但我们也期待智能设备品牌能够从一开始就为他们的客户做正确的事情，并确保购物者能够轻松找到关于其设备支持期限及做出知情购买的信息。”

英国立法的一个[世袭贵族](https://www.parliament.uk/site-information/glossary/hereditary-peers/)Viscount Camrose，被政府任命为网络部长，他说：“随着日常生活越来越依赖于连接设备，互联网带来的威胁倍增，变得更加严重。”

他说：“从今天起，消费者将更加放心，他们的智能设备不受网络犯罪分子的侵害，因为我们将引入世界首个法律，确保他们的个人隐私、数据和财务安全。”

尽管其他地方也在推进类似的法律，但尚未生效。欧盟的[网络韧性法案](https://therecord.media/eu-proposes-security-standards-for-iot-products)尚未最终达成协议，但其类似条款预计将在2027年前不适用于欧盟内部。

在美国，尽管[2020年物联网安全改进法案](https://www.congress.gov/bill/116th-congress/house-bill/1668)要求国家标准与技术研究院“制定和发布联邦政府如何使用物联网设备的标准和指南”，但并无联邦法律关于保护消费者物联网设备的规定。
