# "Roku在最新的安全疏失之后为所有用户启用了双因素认证。• The Register"

> "来源：[https://www.theregister.com/2024/04/15/roku_2fa_for_everyone/](https://www.theregister.com/2024/04/15/roku_2fa_for_everyone/)"

"流媒体巨头 Roku 在今年早些时候的一次攻击中，攻击者访问了约 591,000 个客户账户后，正在使双因素认证（2FA）成为强制措施。"

"这是 Roku 在时间轴上详细说明的内容，但它说这些妥协发生在两次单独的事件中。第一次影响了 15,363 个账户，导致公司从三月开始更加密切地监控账户活动，而这种监控导致了发现第二次事件，影响了大约 576,000 个账户。"

"在不到 400 个案例中，攻击者使用这些账户购买了流媒体订阅和 Roku 硬件，使用了用户账户中存储的付款详细信息，公司确认。"

"Roku 表示，这些账户持有者中的每一个都已经得到了赔偿，并且在任何情况下，攻击者都无法访问任何敏感信息，包括“完整的信用卡号码或其他完整的支付信息”。

"根据在美国于 3 月 8 日发布的数据泄露通知书，社会安全号码、出生日期和其他类似信息也没有受到影响。"

"Roku 还表示，其系统似乎也没有受到妥协。公司认为，这些账户是通过凭据填充攻击访问的，攻击者使用了从其他来源窃取的凭据。"

"凭证填充和密码喷洒都是相当相似的暴力攻击类型，但前者使用已知的凭据对（用户名和密码）。后者只是在已知用户名上垃圾邮件常见密码，希望其中一个会导致已认证的会话。"

"在更新给客户的声明中，[它](https://www.roku.com/blog/protecting-your-roku-account)表示，没有迹象表明 Roku 是这些攻击中使用的账户凭据的来源，也没有 Roku 的系统在任何一起事件中遭到损害。"

""相反，这些攻击中使用的登录凭据很可能来自另一个来源，比如受影响用户可能在另一个在线账户上使用了相同的凭据。""

"所有账户现在要求实施[双因素认证（2FA）](https://www.theregister.com/2024/01/11/mandiant_x_account_brute_forced/)，无论它们是否受到了大波妥协的影响。Roku 有超过 80 百万活跃账户，所以只有少数受到影响，这些账户都已经强制重置了密码。"

"不论是否受到影响，所有用户都被鼓励为他们的账户创建一个[强大而独特的密码](https://www.theregister.com/2023/11/20/your_password_hygiene_is_still/)，至少包含八个字符，包括数字、符号和大小写字母的混合。"

"英国的 NCSC 仍然建议创建由三个随机单词组成的密码，其中一个足够长且强大以满足复杂性检查的要求。"

"这突显了使用密码管理器与独特密码的必要性，" Pentest People 的首席顾问 Josh Hickling 说道。 "如果网站的用户没有重复使用他们的密码，他们就不会受到影响。

"借助于 Apple Keychain，或者像 1Password 或 LastPass 这样的第三方应用在安卓上，管理一系列独特的密码变得比以往任何时候都更加容易。这有助于减少您的密码在第三方网站上泄露的风险，并影响到您在不同系统上的使用。"

Roku 也要求用户保持警惕，防止其服务遭遇可疑活动，例如钓鱼邮件或点击可疑链接以重置密码——常见的问题。

"最后，我们对这些事件的发生表示诚挚的歉意，以及它们可能带来的任何影响。" 它说道。 "您的账户安全是我们的首要任务，我们致力于保护您的 Roku 账户。" ®
