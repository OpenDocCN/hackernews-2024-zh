# 合成器之争：MIDI的故事，一统音乐世界的接口 | MusicRadar

> 来源：[https://www.musicradar.com/news/the-story-of-midi](https://www.musicradar.com/news/the-story-of-midi)

**如今几乎无法想象在音乐技术领域存在硬件和软件不兼容的世界。**

想象一下只能在某种型号的计算机上运行的[数字音频工作站（DAW）](https://www.musicradar.com/news/the-best-daws-the-best-music-production-software-for-pc-and-mac)或插件。或者只能通过特定昂贵设备盒子对合成器进行序列化。

但这正是80年代初的世界，合成器开始逃离恐龙摇滚的领地，成为非音乐人的选择武器。

[**Dave Smith：合成器先知**](https://www.musicradar.com/news/dave-smith-synth-prophet)。“传奇”可能是一个过于耐用的词汇，但当我们将这个描述用于合成器先驱Dave Smith时，毫无争议。在这里，我们追溯他具有影响力的职业生涯……

在60年代，[Moog](https://www.musicradar.com/tag/moog)开创了合成器的发明，到70年代末，无数公司都在竞相使技术更便宜，体积不再像衣柜那么大。ARP、Obeheim和Sequential Circuits等公司，更不用说日本的[Roland](https://www.musicradar.com/tag/roland)、Korg和[Yamaha](https://www.musicradar.com/tag/yamaha)等推出的新款更加经济实惠的合成器……

Moog为控制其合成器创造了CV/Gate系统。CV/Gate通过电压发送到Gate以打开它（允许声音传播），然后切断电压以再次关闭它。

与此同时，不同的电压作用于CV（控制电压），控制音调。电压越高，音符越高。他们的键盘模块通过CV/Gate来播放合成器，而他们内置的序列器则省去了手动演奏的必要，让机器自行触发。

> 很快其他合成器制造商意识到，创造一款“自弹自唱”的合成器可能不仅仅是合成器发烧友的新奇玩意。毋庸置疑，这将是一场大事……

很快其他合成器制造商意识到，创造一款“自弹自唱”的合成器可能不仅仅是合成器发烧友的新奇玩意。创造可编程的序列器实际上可以解放音乐，第一次将作曲行为从演奏乐器的能力中解放出来。毋庸置疑：这将是一场革命。

对于所有这些新兴合成器公司来说，问题在于，虽然他们都在同一时间想到了同一个想法，但他们却以不同的方式实现了乌托邦式的“自弹自唱”愿景。

Moog 的 CV/Gate 系统是一个直截了当的解决方案。它使用了一个简单的系统，其中一伏特代表一个八度。因此，1V 会播放 C1，2V 会播放 C2。每个整伏特之间有12个电压级别，覆盖了所有中间音符。简单明了。

Roland 的早期合成器（System-100）和 Sequential Circuits 的 Prophet-5 依靠 Moog 的标准，因此享有一定的兼容性。然而，即使是它们也需要精确调谐和电压的仔细衰减，以确保无缺陷的播放。

即使是同一制造商的合成器也各有各的问题。例如，Moog 自家的 [Minimoog](https://www.musicradar.com/news/vintage-music-tech-icons-moog-minimoog-model-d)，在与其他设备一起使用时，以其神秘莫测而出名，电压会在一个合成器上触发 C 音，但在 Mini 上触发 F 音。

与此同时，为了追求更低的成本，像 Korg 和 Yamaha 这样的厂商追求了一种更易于制造和更工匠式的系统，其中电压加倍就加倍了八度。额外的功率和更大容量的步骤导致了更少的挑剔性播放，但与其他系统的兼容性也有所降低。

随着 CV/Gate 的世界变成了一个无休止的微调噩梦，更适合电子工程师而不是音乐家，每个制造商都力图创建最佳、最可靠、最‘不易调整’的接口，并在竞争中获胜。

Roland 在 1981 年采用了其 DCB（数字控制总线）系统。它首先出现在其 [Juno-60](https://www.musicradar.com/news/vintage-music-tech-icons-roland-juno-60) 和后来的 [Jupiter-8](https://www.musicradar.com/news/blast-from-the-past-roland-jupiter-8) 系列中。这一专有系统通过提供对放大器和滤波器的控制，以及简单音符值的控制，胜过了竞争对手。如果您想在旁边运行鼓机，也不是问题——Roland 还为此发明了一个接口。DIN-Sync 使用 5 引脚 DIN 电缆（在日本高保真组件中很流行），因此您的 MC-4 Microcomposer 可以与您的 [TR-808](https://www.musicradar.com/news/roland-tr808-synth-icons-best-tracks) 鼓机并行运行。

[Oberheim](https://www.musicradar.com/tag/oberheim-electronics) 则选择了其（当时非常令人印象深刻的）37引脚 D-Sub 连接器，创建了‘The Oberheim System’。这使用户能够将他们的 DX 或 DMX 鼓机和最多八台 [OBX, OBXa 或 OB-8](https://www.musicradar.com/news/tech/synth-icons-oberheim-ob-series-597902) 合成器连接在一起，并从他们的 DSX 连续器中序列化所有设备（能够运行八个不同的部分并存储 6000 个音符）。

Sequential Circuits、Moog 和 Korg 眼巴巴地望着，他们的 Model 800、Moog 960 和 Korg SQ-10 还停留在 CV/Gate 阶段。但最终，他们都意识到，CV/Gate 的时代已经一去不复返了，因为一些技巧，如能发送补丁变化和记录旋钮动作与音符一起的能力，很快就会让它们全部过时。

尽管创造了 DCB 并且 - 至少暂时 - 胜过了他的竞争对手，Roland 总裁掛橋郁太郎甚至有更大的想法。迄今为止的常规智慧是通过创建自己的标准来锁定观众；如果他们喜欢你的合成器，那么他们必须也购买你的音序器和鼓机来使用。

实际上，虽然有些音乐家愿意为完整、匹配的单一制造商“系统”支付数千美元，但大多数用户却远离一切，害怕在80年代的创新如山倒海般涌现时投资错误的产品。

掛橋意识到，如果摆脱购买焦虑，音乐家们可以购买他们想要的东西（并且更多的东西），确信他们的工作室将在未来几年“正常工作”。因此，他联系了 Oberheim 公司的 Tom Oberheim，讨论他们如何共同避免即将到来的标准战争，这将导致他们所有人不断创新，最终走向灭亡。

Roland 创始人**掛橋郁太郎**（2000年）（图片来源：The Asahi Shimbun/Getty）

虽然 Oberheim 对其系统非常满意，并且得到了许多专业音乐人的青睐，他建议掛橋与 Sequential Circuits 的 Dave Smith 联系，因为 Smith 已经在制定计划了。

Smith 想象中的世界，所有设备都使用相同的数据格式 - 就像它们都使用相同的电源电缆和插头一样。

作为1977年 Prophet 5 的创始人和首席设计师 - 世界上第一台数字可编程多音合成器 - Smith 创造了一个传奇。这款声音出色的模拟合成器具有即时的数字存储和召回功能，自然而然地以千计的销量使他的公司名声大噪。

更重要的是，它的继任者 Prophet 10（基本上是两个 Prophet 5，带有双键盘）配备了内置数字音序器，每隔几微秒记录其两个五八度键盘上每个键的状态，并能够回放表演。 Smith 公开了这项技术，希望其他公司采用这一标准，并为他的 Prophet 合成器制作音序器，从而扩展其吸引力。

不幸的是，该系统数据量过大，未能起飞。相反，Smith 开始研究一个更简单的系统，不再每隔几微秒报告每个键的状态，而只是简单地记录按键和松开的音符。

随着 Oberheim 的加入，Smith 终于在掛橋身上找到了一个盟友。现在，他们所要做的就是让整个合成器行业同意他们的观点。

Smith将他的系统命名为通用合成器接口，于1981年10月在Gakki音乐展上，与Ikutaro Kakehachi的介绍一起，向Yamaha和Korg的代表们展示了它。它受到了良好的反响，但其问题也显而易见。Korg指出，使用插孔传输所需信号会在系统中产生噪音。此外，还没有同步音序器到鼓机的条款，或者让一台机器启动和停止另一台机器的方法。

Yamaha建议添加一个接地针来解决噪音问题，而Roland提出了一种控制速度并发送启动和停止消息的方法。很快，在彼此同意USI缺少什么之后，一个新的标准开始形成，而日本的代表团开始使用UMI（通用音乐接口 - 发音为You-Me）这个名称，Smith则提出了音乐仪器数字接口（类似发音为MIDI），以更好地传达其时尚全数字化的特性。

几个月内，Smith吸收了合作伙伴的想法，并建议在1982年1月在美国举行的年度NAMM展会上与合成器界的伟大人物进行更广泛的讨论。每个合成器制造商都将在城里 - Moog，ARP，Yamaha，Roland，Korg，Voyetra，Syntauri，Octave Plateau - Smith找到了一个愿意倾听的观众，即使只是为了弄清Sequential Circuits接下来有什么热门新机器。

在会议上，Smith概述了他的提案。与其各自开发和构建不同的序列标准，销售越来越少，为什么他们不都同意一个格式，并在未来多年中共享所有的好处呢？

MIDI将是一个数字系统，分辨率为128个值（足以涵盖任何参数设置），通过16个通道广播（因为谁有超过16个合成器？），具有‘运行状态’，可以启动和停止其他设备，并使所有设备同步运行。所有内容可以通过接地的、无噪音的5针DIN电缆发送，这已经被Roland用作其DIN Sync和流行的廉价高保真连接器。

> 但会议并不顺利... 许多大公司已经开始研发更复杂的系统，允许更精细的控制。

但会议并没有进展顺利。虽然普遍的共识是数字化是未来的方向，Smith的乌托邦愿景 - 设计得尽可能用户友好和快速修复 - 因其简单性而受到了嘲笑。Smith的新接口缺乏分辨率和通道的限制没有给任何人留下深刻印象。

许多大公司已经开始研发更复杂的系统，允许更精细的控制。一些制造商要求无限数量的合成器。有些要求音频从一个设备数字传输到另一个设备。甚至有人质疑是否应允许用户购买其他品牌的合成器。随着辩论毫无进展，观众们散会了。

但是 Smith 与 Roland 和 Yamaha 的对话仍在继续。此时，大部分工作已经在 Sequential 在旧金山的办公室完成，并将文件发送到日本进行批准。至于行业的其他部分呢？那有什么关系呢？Smith 知道，如果 Sequential、Yamaha 和 Roland 这些巨头能够创建一个标准，全球的合成器制造商都会前来乞求加入。

因此，整个1982年，工作依然在进行中。令人惊讶的是，这三家公司从未交换过硬件，也没有互访，也没有进行任何形式的测试。相反，这三家公司的工程师们达成了规范的一致，然后开始将其实施到他们的下一代合成器中。

（图片来源：Reverb / Billy Corgan）

对于 Sequential Circuits 来说，那款合成器就是 [Prophet-600](https://www.musicradar.com/news/vintage-music-tech-icons-Sequential-Circuits-Prophet-600)。这款六音（从五音增加到六音）的 Prophet-5 续集通过更多的数字芯片降低成本，但最终因其更加严厉的声音而未能打动人心。然而，正如 Smith 的梦想一样，这款合成器在1982年12月发布时成为第一款配备 MIDI 端口的合成器。

Roland 的第一款 MIDI 合成器是 Jupiter-6。从 Jupiter-4 进化到 Jupiter-8 已经是一个重要的一步，增加了多音和 VCOs（从每个音道一个增加到两个），而Jupiter-6 则是填补空缺的练习。昂贵的 Jupiter-8 已被 Roland 自己的 Juno-6、60 和 106 所取代，留下了足够容纳六音多音的‘baby 8’的空间。

值得一提的是，可能是由于对 MIDI 的发展缺乏信心，‘6’ 跳过了标准的 MIDI Thru 端口，因为 Roland 的工程师们急于控制成本，无法想象有人会拥有超过两台 MIDI 设备的情况......

而与此同时，Yamaha 的第一步进入 MIDI 领域是 [DX7](https://www.musicradar.com/news/vintage-music-tech-icons-Yamaha-DX7)，这款开创性的全数字 FM 合成器在整个80年代凌驾于其模拟同行之上。用模拟 CV/Gate 控制数字内部将是一个昂贵的接口难题，因此当 MIDI 的概念在项目接近完成时出现时，简直是天赐之物。

然而也存在问题。由于赶工完成 DX7 的设计，导致‘非关键功能’被放在次要位置，因此早期的 DX7 没有办法改变广播的 MIDI 信道。它只能使用 MIDI 信道 1，否则无法作为控制键盘。

此外，由于 MIDI 规范在后期才出现，选择的键盘并不能准确地产生 MIDI 的 127 个不同速度级别。通常情况下，DX7 发送的范围在 16 到 116 之间。如果你很幸运的话。

而合   当合成器巨头们享受着新的合作时代时，史密斯却不想透露太多关于未来的计划。史密斯确信未来在于多音色合成器——能够同时演奏多个声音的合成器，因此在MIDI中内置了‘单音模式’，使得一台合成器能够播放多个声音。

然而，史密斯并没有将其标榜为如此，也没有提醒他的竞争对手即将到来的MultiTrak合成器，反而将其偷偷地写入了规格中，导致雅马哈将‘单音模式’解释为使整个合成器变为单音的开关。

也许最具象征意义的是，MIDI是DX7的一个如此晚的添加，以至于其最小化的（且已设计好的）控制面板完全没有提及它。为了调整DX7的接收频道（幸运的是是可编辑的），用户必须通过按下Function/Space然后Button 8（标记为Feedback）来访问它，作为一个隐藏特性…

因此，当MIDI在1983年1月的NAMM展会上在Sequential Circuits的展位上首次向世界展示时（正好是一年之前史密斯的提议被驳回的日子），系统能够正常工作几乎是一个小小的奇迹。

Sequential的音效设计主管约翰·鲍文回忆说，他当然测试过连接到另一个Prophet-600的Prophet-600，但无法接触到尚未公布的Jupiter-6。并且，通过紧凑的1982年发货日期，Prophet-600甚至在DX7宣布之前就已经发货了。

然而，当Roland的架八带着新款Jupiter-6（在展会上首次亮相）来到Sequential的Prophet-600旁，并将其连接起来时，两者在聚集的记者面前无缝合作。你可以在一个合成器上演奏并触发另一个合成器的声音，这是一个令人惊叹的创新——一个真正的首创，使得音乐在国际上成为新闻。

但或许史密斯和架八的最大创举是在同意将MIDI定为开放标准。生产MIDI设备不涉及任何费用。该规范是公开的，刚成立的非营利组织国际[MIDI协会](https://www.midi.org/)开始传播这一消息。

一年之内，正如史密斯预测的那样，几乎所有的制造商都加入了进来，即使你连一个5针DIN电缆都没有，MIDI的标准和协议可能仍然在你的录音室幕后默默运行，赋予创作者力量，并确保每一件音乐硬件和软件之间的兼容性。
