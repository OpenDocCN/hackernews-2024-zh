# Staff UI Engineer | Streak

> 来源：[https://www.streak.com/careers/staff-ui-engineer](https://www.streak.com/careers/staff-ui-engineer)

‍

[Streak](https://www.streak.com) 是建立在Gmail上的CRM系统。我们是一支遍布北美的**远程优先**团队，由35人组成。我们正在快速成长并且非常**盈利**，拥有热爱我们产品的客户。目前，我们正处于产品市场契合的黄金时期，拥有实际收入，同时拥有一个真正扁平化的层级结构，使您能够快速交付产品。

我们希望加快产品交付速度（有很多需要构建的东西！），因此我们正在寻找具有**高能动性**和**经验丰富的前端工程师**，来开发具有高影响力的产品功能/前端基础设施。我们正在建设一个小而专注、高效的前端工程团队，直接与首席执行官一起领导产品。为了帮助打造这支优秀的团队，我们提供极具竞争力的薪酬（基于旧金山地区水平）和一些有趣的[福利](https://www.streak.com/careers#benefits)。我们希望您在这里能做出您职业生涯中最出色的工作，因此您应期待高度的自主权和责任感。

最后，我们认为传统的面试流程与工程师的产品交付能力关联不强。相反，我们的面试流程模仿了实际工作情况。这是高度异步的，偏向书面表达，并涉及构建有用的东西。我们认为这将为您提供对我们团队和公司的深刻洞察，并当然也让我们评估您是否能在这里蓬勃发展。

### 您的技能

1.  **具有高能动性和紧迫性** → 您能够独立取得进展，并能够在庞大的代码库中导航（不用担心，我们的团队乐意在您遇到困难时提供帮助）。面对大竞争对手，我们的区别在于产品速度，因此您默认偏好紧急交付。

1.  **具备一流的异步/书面技能** → 远程协作对我们尤为重要，因此您应具备出色的书面和实时协作能力。如果您默认选择首选异步沟通，那么您在这里将表现出色。我们经常编写设计文档，在会议前总结观点，记录我们的代码或调试过程的Loom视频。

1.  **是前端专家** → 您应具有5年以上使用React和TypeScript等现代前端工具的经验。如果您曾经为现代Web应用程序的用户界面功能发布过，那么您将是一个非常合适的人选。用户不必是百万级，但足以涵盖所有边缘案例。

1.  **付出努力** → 我们不会加班来完成任务，但我们确实致力于每周保持一致的40小时专注工作。这应该足够完成出色的工作并推动项目完成。专注是通常使“几乎完成”与“已发布”之间的区别。

1.  **设计解决方案** → 这里没有架构师。每一个像素和程序状态都没有详细说明。你应该能够设计一个与我们的代码库良好配合的解决方案。在产品规范中填补空白并在需要时做出产品决策。

1.  **务实的态度** → 你应该能够在新功能、重构、重写和使某些东西正常工作之间取得平衡。你还能识别出何时值得投资新事物。你将人工智能视为放大我们努力的工具，而不是竞争对手。

1.  **提升团队水平** → 你愿意通过审核PR、撰写优秀的文档并演示你的工作进展来与其他工程师分享你的专业知识。你通常是一个乐观主义者，喜欢与他人合作。

### 你将会负责的工作

在Streak，产品工程师在全栈上构建用户界面功能，重点放在前端技术上。我们致力于两个产品，[Streak](https://www.streak.com) 和我们的**开源** [**InboxSDK**](https://www.inboxsdk.com)（用于在Gmail内构建应用程序的开发者工具，当然我们自己也在使用）。

你将从头到尾负责项目，并负责发布用户喜爱的高质量软件。以下是一些有趣的最近和即将面临的技术挑战的例子：

+   **高性能电子表格** → 我们的主要UI是协作实时电子表格。它需要具有高性能（60 fps）并支持数百万个不同类型的单元格。

+   **第一方风格的Gmail集成** → 我们整合Gmail，使我们的Streak集成看起来非常本地化。我们在Gmail的整个UI中进行集成 - 在撰写窗口、左侧导航、搜索框、线程工具栏、右侧边栏、消息视图等。你将为我们广受欢迎的开源[InboxSDK库](https://github.com/InboxSDK/InboxSDK)和[其他](https://github.com/StreakYC/page-parser-tree)做出贡献。

+   **数据同步架构** → 我们希望在Streak中的互动感觉自然。在本地数据模型、缓存、内存优化、渲染优化等方面有大量工作。特别是考虑到在类似Gmail这样的重量级应用程序中运作的复杂性。

+   **人工智能** → CRM 是人工智能的完美应用案例。我们利用庞大的数据集（完整的销售管道，公司的每个电子邮件收件箱等）+现成的LLMs，为我们的用户消除所有繁琐的工作。我们已经发布了几个基于人工智能的功能，以及自行开发的基础架构，以便快速迭代。

+   **组件系统** → CRM 有很大的产品表面积。我们需要能够快速创建 UI，并确保其与我们应用程序的其余部分具有一致的外观、感觉和行为。在此过程中，我们面临一些有趣的挑战，包括何时以及如何 conslidate 不同世代的组件。

+   **开发、构建、测试和部署工具** → 鉴于我们的应用是一个扩展应用，这里有一些独特且具有挑战性的要求。我们在构建/CI 时间、类型检查和增量部署系统方面投入了大量精力。这些都受到浏览器特定技术和政策要求的复杂影响。我们与 Google 的 Chrome 扩展团队密切合作。

+   **精美的数据可视化** → 仪表板不是开玩笑的。如何在允许用户根据他们的需求定制报告的同时制作美观的数据可视化，而不需要重新构建各种 BI 工具的所有 UI？（提示：使用一点点 AI 的技巧）。

+   **漏斗优化** → 由于大量资金通过 Streak 流动，因此优化各种漏斗和用户流程具有很高的影响力。目标是在提升用户体验的同时，更多用户转化到我们更高级别的计划。

### 我们的工作方式

在工程团队中，我们遵循 [ShapeUp](https://basecamp.com/shapeup) 方法论。我们将项目划分为 2 或 6 周的“需求量”。与其估计某件事需要多长时间，这些需求量是我们将在功能上花费的严格时间界限。如果在此期间内无法交付，我们不会自动继续工作。这鼓励我们首先构建一些最小但高质量的东西，以确保至少交付给用户有价值的东西。作为工程师，你可以灵活地减少范围，以确保交付。这使我们能够每六周向用户持续发布功能，并避免长时间的项目。如果你想要跟踪其中一个项目的进展，[我在推特上](https://twitter.com/aloo/status/1386777105405419521) 发布了整个功能的最终结果。

在周期之间，我们通常会冷静下来，处理错误、重构等工作。我们也利用这段时间来规划下一个周期。功能规格有严格的界限，但内部可塑性强。这鼓励工程师在构建过程中调整范围，以确保在周期结束前交付成果。我们避免制定非常具体的长期路线图。相反，我们有一个大致的前进方向，但每六周，我们会重新评估接下来的计划。

我们是为那些想要专注构建和交付而没有干扰的工程师量身定制的完美规模。我们更倾向于由热爱自己工作的小团队组成，而不是那些管理开销较大的大型低效团队。

### 薪酬

该职位的总薪酬为$175,000（高级）- $400,000（高级员工）。其中包括基于个人和公司表现的10%奖金部分。薪酬（基础工资和奖金目标）每6个月进行一次调整，与绩效评估同时进行。随着工作年限的增加，奖金目标会大幅提升，这并不罕见，你的奖金目标可能会成为总薪酬的重要组成部分。与基于股票的薪酬不同，奖金非常流动，并且每6个月支付一次。
