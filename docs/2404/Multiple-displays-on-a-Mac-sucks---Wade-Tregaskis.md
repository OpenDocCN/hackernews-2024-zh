# Mac 上的多显示器支持真的很糟糕 – Wade Tregaskis

> 来源：[https://wadetregaskis.com/multiple-displays-on-a-mac-sucks/](https://wadetregaskis.com/multiple-displays-on-a-mac-sucks/)

我与许多人共事过，他们使用多显示器 – 通常是一对匹配的并排显示器 – 并对此赞不绝口。说实话，这总是让我感到困惑。

在过去的三十年里，我曾几次尝试多显示器生活方式。起初是作为一种资产阶级的放纵，甚至拥有单一的17英寸显示器都被视为奢侈^([1](#f3ad31a4-54b3-4c05-9f1f-e3057f4f8a73))，更不用说两个显示器了。然后，偶尔在多年之后，更实际地尝试扩展我的屏幕空间。

2014年5k显示器的首次亮相长期以来减轻了压力。很难过分强调第一台5k iMac的深远影响。

但是在过去的一年左右，我感到有点局促 – 尤其是因为我在Xcode中花费的时间越来越多，虽然在5k显示器上还可以，但并不理想。特别是当你尝试进行GUI设计时，无论是AppKit还是SwiftUI。更不用说如果你开始使用iPad Pro模拟器之类的东西。

所以我再次考虑了我的选择。

可悲的是：

1.  我有一台iMac Pro。即使我愿意浪费内置显示器，我也不能这样做 – 苹果不允许iMac关闭它们的内置显示器。

    我不一定介意替换iMac Pro（自从制造之日起它就是一台相当糟糕的电脑），但[苹果一直在密谋阻止我这样做](https://wadetregaskis.com/apples-timing-problem/)^([2](#2d929093-1c10-4680-a4bd-383b85a98e88))。

1.  没有更大显示器的好选择。

    目前的6k显示器并不理想^([3](#85221166-1dce-4d95-b6be-6b7fa2b7c644))，尽管我很高兴至少有几个还算可以的选择。我从未想过我会感激*戴尔*这样的硬件公司仍然存在并且比苹果做出更好的产品。

    8k显示器实际上几乎不存在 – 我没有足够的物理空间使用8k电视^([4](#31755725-c69f-4729-b5d0-d0e1d2833f9f))，否则这实际上是最佳选择，而我知道的唯一一款8k电脑显示器太小了，分辨率太高^([5](#329e19a1-f614-448f-8975-7365a698f75a))。

突然间，我从在LinkedIn工作时剩下了一台5k显示器（当时他们不会为员工提供体面的显示器，所以我不得不自己掏腰包购买^([6](#3f8979d6-ea6a-4991-afc8-845186aeb8a5))）。就多显示器设置而言，这个显示器可能是理想的，因为它基本上是iMac Pro显示器的完全相同面板，所以它应该非常匹配^([7](#9cf23db5-75b6-40a4-88cd-48d1ad743001))。所以几个月前，我花了一些精力重新布置我的工作空间来容纳它，并开始使用它。

三十年来，变化不大。

## macOS操作系统*仍然*在重新启动应用程序时非常糟糕地记住窗口的位置。

在最初的一个月左右，情况还算不错 —— 有几个应用程序*偶尔*会有问题，但还可以忍受。特别是Xcode总是让我恼火，因为每次打开它时，它*总是*忘记了文档窗口的位置^([8](#08475590-6d49-4fed-ab6b-99455053ae2a))。

然而，随后发生了对我来说看不见的事情，自那时起*每一次启动，每一个窗口都会重置到内置显示器的位置*。每次启动我都必须花费大量时间从拥挤的主显示器中找出被埋藏的窗口，并将它们移到第二显示器上的正确位置。

我本可以将此归咎于特别的倒霉运气，但几十年的经验告诉我，这实际上是很正常的。它*总是*会发生，迟早都会。而且在某个时候它会自己解决，并[大多数情况下](https://wiki.example.org/mostly)记住窗口的位置。只是不可避免地重复这种恶性循环。

## 全屏应用程序（主要是游戏）不能正确处理额外的显示器。

这两种情况都有 —— 有些应用程序错误地忽略额外的显示器，而有些错误地*不*这样做，并做出愚蠢的事情，比如把窗口放在错误的显示器上：

+   游戏是前一类中最常见的违规者，因为它们通常无法调暗或关闭未使用的显示器。当你尝试玩游戏时，身边有一个明亮的显示器非常分散注意力。每次我启动或退出这样的应用程序时，我都必须手动打开系统设置，进入显示设置，点击第二显示器，并手动调整其亮度。

    我的第二显示器（[LG 5k](https://www.lg.com/us/monitors/lg-27md5kl-b-5k-uhd-led-monitor)）没有电源按钮，我很快发现，拔掉雷电线会使macOS陷入困惑，导致窗口乱动，有些应用程序直接崩溃。

+   在后一种“你在干什么？！”的子类中，有些应用程序像Adobe Lightroom Classic，有时会自动将整个窗口移动到另一个显示器上，唯一的办法是退出并重新启动Lightroom^([9](#652e1ed2-b22f-476a-bfcf-9c76338c27b3))。

## 侧边Dock不实用。

当Dock位于侧边时，它只会出现在靠那一侧最远的显示器上。必须在*多个*显示器之间移动鼠标才能到达Dock，这种速度和方式都无法接受。

我把Dock设置为自动隐藏，因为我不是一个怪物。但这意味着我必须小心靠近屏幕边缘的鼠标移动。

我更喜欢将Dock放在左侧，因为这是最不显眼的位置——在左边的鼠标移动和交互相对较少（右边是次佳位置，但显著的是滚动条的所在地）。而且鼠标向左（或向右）移动稍微比向下移动容易，因为向下移动（朝向我）会因为USB线缆的张力增加额外的压力。

使用多个显示器（典型的水平布局），我被迫使用底部的Dock，这是最糟糕的位置，因为屏幕底部有太多东西——窗口调整大小的小部件、工具栏及其按钮、水平滚动条等等。必须精确地移动鼠标几个像素以避免触发Dock，同时仍然需要将光标移动到屏幕边缘附近。有时候必须移动整个窗口才能与其底部进行交互——这可能需要先从顶部调整大小，以便允许向上移动。

我把菜单栏设置为自动隐藏，部分原因是为了视觉整洁和节省一些屏幕空间，但也因为菜单栏曾经在我的前一台iMac上烧屏，非常糟糕。

我也经常使用菜单栏，不仅用鼠标激活菜单，还在其中安装了iStat Menus和许多其他监控工具。所以我经常需要把鼠标移到屏幕顶部显示菜单栏。这本身不是负担——把鼠标移动到屏幕顶部完全是一个微不足道的操作。

但当你使用多个显示器时，它却不起作用一半的时间，因为菜单栏只会在“活动”显示器上显示自己，而这往往在视觉上与“非活动”显示器一样难以区分（感谢苹果长期以来对清晰度的战斗）。

这违背了本能和肌肉记忆，即只需轻松将光标移至显示器顶部即可查看菜单栏。更糟糕的是，由于macOS在菜单栏显示方面存在一些设计缺陷（例如，如果最前端应用程序的主线程运行循环未空闲，或者应用程序正在启动时，它会拒绝显示），你不能立即确定它是否因为光标在“错误”的显示器上而拒绝显示自己。因此，总会有浪费时间、分散注意力的瞬间，可能需要移动到完全不同的显示器。对于每天发生数百次的操作来说，浪费的几秒钟会累积起来。

## 有时候窗口会出现在错误的显示器上。

其实不用多说了。我不知道为什么它们会出现在错误的位置，但macOS的窗口管理有很多类似这样无法解释的bug，比如为什么有些模态对话框会出现在所有现有窗口的*后面*。

## 多显示器的人体工程学往往很糟糕。

在典型的水平排列中，如果你同等重视两个显示器的位置，你会发现一个在你的左边，一个在你的右边，它们之间有一个非常恼人的间隙，而这个间隙恰好是你自然而然要看的地方——直视前方。键盘或鼠标都不与任何显示器对齐，我发现这会影响打字和鼠标操作的准确性（特别是在我不停地移动的跑步机桌上）。而且无论如何，我总是会感到脖子酸痛，因为我花了所有时间都在看中心以外的地方。

如果你把一个“主”显示器放在前面中央位置，并将第二个显示器放在一侧，那么第二个显示器就会变得*远不如第一个有用*，因为实际上看它会更加不舒服。这也导致了这样一个事实……

## 在多个显示器之间分割工作是*困难的*。

实际上，两个窗口并排在一个显示器上与两个窗口分别在不同显示器上之间有着*很大*的区别（尤其是如果这些显示器不是相同的，或者它们在水平或垂直轴上不对齐）。也许这就像是移动你的眼睛和移动你的头之间的简单区别。

无论是什么原因，我都无法频繁地在不同显示器之间舒适地切换。这迫使我不将任何*重要的*东西放在第二显示器上（特别是如果它的人体工学位置比主显示器差的话），所以它最终成为了我不经常使用的东西的一种储存地点——例如日历、剪贴板查看器、计算器等。

由于我实际上并不经常使用它们，所以将它们放在第二显示器上与仅在主显示器上将它们置于前台然后完成后再隐藏它们相比，实际上没有太多优势。事实上，有时候这是适得其反的，因为第二显示器上的无关动画会分散我的注意力，而如果它们安全地隐藏在主显示器后面的其他窗口后面，我则不会分心。

到目前为止，我唯一真正需要同时使用的有效用例是将信息参考（例如Xcode的文档窗口）移动到第二个显示器上。事实上，这是我决定费事地连接第二显示器的具体用例，因为在一个半5k显示器上几乎不可能容纳一个复杂的Xcode项目，而且不断在代码和文档之间切换是令人烦恼和稍微分散注意力的——当然，如果它们同时可见，这样做会更加容易。

## 后续

在HackerNews上有一个[关于这篇文章的令人惊讶的长帖子](https://news.ycombinator.com/item?id=40166268)。而我惊讶地发现，大多数人都在诉苦并真诚地试图帮助，而不是充满恶意。如果你想深入了解这个话题，看看这个帖子是值得的。

HackerNews评论中提到的一些要点：

+   我使用Mac已经有大约三十五年的时间，一直专注于Mac。因此，断言我“做错了”的说法，因为我实际上是个Wintel或Linux的傀儡是错误的。🙂

+   很多人也提到甚至有困难使外接显示器可靠工作，这点我深有体会——说实话，我使用的那个LG 5k以前真的是个大麻烦，后来才发现它附带的雷电线有问题。即便如此，它在我的工作MacBook Pro上始终无法可靠工作，原因不明。但似乎在我的iMac Pro上运行得很好。🤷‍♂️

    我选择在这篇文章中不深入讨论这个问题，因为目前（🤞）我似乎并没有遇到这种问题，而且因为我觉得这是一个完全*不同*的大问题。但是，对于Mac来说，这绝对是一个*大*问题。

+   有一两个人建议我的问题在于我应该只用一个更大的单屏显示器。我同意这一点。这在帖子的开头已经讨论过了。🙂

    相当多的人建议或至少宣扬使用超宽显示器。如果有一款像素密度合适的超宽显示器发布的话，我会考虑的。例如，5k是一个在23.5英寸（传统27英寸显示器的水平宽度）上的合理像素数量，但在像50英寸宽的显示器上*远远*不够。太少了。我在文末的脚注中略微提到了这个话题，但再次强调：一个合格的显示器在合理的工作距离上，像素密度*至少*达到100 PPD（每度像素数）。苹果的“Retina”显示器都符合这一标准。据我所知，迄今为止没有任何一款超宽显示器达到这个标准，甚至还差得很远。

+   我最近使用的具体设置是我的iMac Pro传统地放在前中央位置，LG 5k则偏向左侧约45°角。

    在以前尝试多显示器生活方式时，我使用过许多其他设置，包括对称的水平布局，堆叠的垂直布局，甚至像iPad Pro或笔记本电脑这样的临时“对角线”布局。我想我已经尝试了所有合理的布局。

+   我不使用[Spaces](https://en.wikipedia.org/wiki/Spaces_(software))。过去我用过它（主要是在Mac OS X刚推出时，以及更早的Linux版本），但它从来没有真正感觉良好，也从来没有正常工作过。它还有自己的窗口管理问题和bug，例如新窗口（特别是模态对话框）悄悄地打开在错误的空间里。😔

+   我不会全屏应用程序（除了非常有限的情况，比如一些*只能*全屏运行的游戏）。

    多年来，我在我的工作 MacBook Pro 上确实如此，但只是因为我感到被小的笔记本屏幕迫使这样做——macOS 的全屏模式有许多*非常多*的bug。对于大多数应用程序而言，当你拥有一个足够大的屏幕时，这几乎没有什么意义，因为你最终会在边距等地方浪费大量的空间。事实上，我对这些bug感到厌倦，即使在我的小12英寸 MacBook Air 上，我也不再使用全屏模式，而是忍受屏幕空间的损失。更重要的是，我现在不再在笔记本上进行实际的软件开发或照片/视频工作——主要是写作和浏览网页。

+   有些人似乎对我使用 Dock 感到惊讶，这反过来也让我感到惊讶。我从未真正考虑过不使用 Dock。

    总结一下，我主要使用 Dock 来：

    +   切换应用程序。

        我当然也使用 ⌘⇥，但是在多个显示器的情况下，应用程序面板出现在哪个屏幕上是不可预测的，这使得使用鼠标变得效率低下（而且仅仅一遍又一遍地按Tab键太慢了，当你打开了数十个应用程序时）。

        我确实使用 Spotlight 来启动和切换应用程序，但我尽量避免用于常用应用程序，因为 Spotlight 非常不可靠：

        +   有时候它花费非常多的时间来显示浮动的文本输入窗口，并且在此期间的按键可能会被悄无声息地忽略或者去到错误的位置。

        +   有时候它就是不工作。我输入一些字母却没有任何反应，或者它只是显示那令人恼火的“索引”不确定进度条。显然稍后再试。

        +   它会随机更改常用字符序列的含义。例如，我可能已经使用“te”代表TextEdit *数月*，然后突然间它决定“te”是某个十五年前的随机PDF文件。这绝对令人气愤和莫名其妙，并且在这种情况发生时似乎没有明显的解决方法。

    +   访问最小化的窗口。我不知道还有其他什么方法可以访问这些窗口。

    +   访问应用程序的 Dock 图标的上下文菜单——通常是强制退出它，但也有一些应用程序在那里放置了重要的功能，通过其他方式并不容易访问。我知道我也可以用 ⌥⌘⎋，我有时也会用，但出于某种原因，我更喜欢通过 Dock 来操作。

    +   监控（例如未读消息数量）。

+   很多人略带批评地指出，Mac 缺乏窗口平铺和手势功能，尤其是那些来自 Windows 的人似乎是这样认为的。这些批评是公正和广泛准确的，但他们似乎忽视了一个事实，即尽管 macOS 没有大量的手势和平铺功能，但确实有一些（例如苹果所称的“[分屏视图](https://support.apple.com/en-au/guide/mac-help/mchl4fbe2921/mac)”）。

    但是，“分屏视图”实际上是全屏模式的一种专业化，有着许多相同的设计问题和bug。

+   比我想象中的有*更多*的显示和窗口管理实用程序 - *许多*人提出了我尚未遇到的建议和推荐（例如，[Stay](https://cordlessdog.com/stay/)，[Display Maid](https://funk-isoft.com/display-maid.html)，[Phoenix](https://github.com/kasper/phoenix)，[Rectangle Pro](https://rectangleapp.com/pro)）。我可能会调查其中的一些，并且如果它们显著改善了情况，可能会发表后续帖子（但请记住，像 Dock 和菜单栏可访问性这样的问题，更不用说人体工程学这样的基础问题，这些实用程序并没有解决）。

+   有一些人基本上回答说：“哦，这些事情对我无所谓”。这样说是公平的，但并不奇怪 - 各有所好。我很高兴看到大多数人并没有试图一概而论地否定我的观点，只是指出还有其他观点。这完全可以理解，并且记住这一点是很重要的。
