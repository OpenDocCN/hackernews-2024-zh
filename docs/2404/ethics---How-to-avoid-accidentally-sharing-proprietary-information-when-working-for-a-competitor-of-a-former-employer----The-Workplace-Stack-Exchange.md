# 道德 - 在为前雇主的竞争对手工作时如何避免意外分享专有信息？- The Workplace Stack Exchange

> 来源：[https://workplace.stackexchange.com/questions/197282/how-to-avoid-accidentally-sharing-proprietary-information-when-working-for-a-com](https://workplace.stackexchange.com/questions/197282/how-to-avoid-accidentally-sharing-proprietary-information-when-working-for-a-com)

[仔细阅读你的保密协议。它会告诉你其中的一些内容。我不能引用我签署的任何协议，因为它们都包括我不能这样做的条款，但详细阅读你自己的协议。还要记住，过于宽泛的保密协议通常会被法院撤销，所以如果你表现得合理，你不会引起法律或道德上的风险。]

假设在工作X的过程中，你学到了一个真实的事实，比如“A比B更快”或者“这些螺丝在X部署的环境中不会生锈”或者“你应该每天磨砺你的凿子来雕刻这些东西”。如果你在工作Y时说“对于X，我们使用A是因为它更快”，你正在揭示关于X的信息，可能违反你的保密协议。但如果你说“在这里A更快”或者“这些螺丝在[海水或其他地方]不会生锈”或者“你今天磨砺了你的凿子吗？”你就没有揭示X的信息。

大多数保密协议没有人们想象中的那么可执行。它们通常也不那么相关。在某事发布之前，它将具有特定功能的信息是重要的和专有的，应该得到保护。一旦发布，这就在新闻发布会上了！是的，其他事情会永远保密，但并不是你所知道的每一小块信息都有资格。

你应该以最高决心保护诸如你的老雇主关于X的未来计划，前同事的姓名和联系方式，X的定价和支付结构，你的同事们的薪水，以及业务的其他“内部细节”。你还应该保密X中突出表现的超酷专有技巧 - 不同的文件格式，压缩技巧，或者别人没有的使其在某种方式上更好，但其机制不为人知的东西。你可以对已经学会的哪个排序算法更快，或者你行业中常用产品的命令行选项等比较镇定。这些不太是关于X的信息，而是你在X上工作时所发展的技能。被“受你对X知识的影响”并不等同于“告诉人们X是如何内部工作的”。
