# 瓶子绘图机

> 来源：[https://vgnotepad.blogspot.com/2024/04/bottle-plotter.html](https://vgnotepad.blogspot.com/2024/04/bottle-plotter.html)

## 瓶子绘图机

我注意到，在小型葡萄酒生产商中，不使用瓶标签是相当常见的。他们通常只用特殊的（**Edding**）油漆笔在瓶子上画一些基本信息。这就是我得到这个想法的方式。每个人都可以得到打印的纸质标签，但瓶子绘图机怎么办。数控机床可以用油漆笔作为绘图机直接在瓶子上绘制标签。并且通过工具更换器，你还可以拥有更多的颜色选择。

我在脑海中有这个想法已经有一段时间了，但是花了很长时间才进行这个心理上的启动并开始工作。通常对我有效的是，当心情适合时，我会出门散步，并完全投入到这种状态中，在那个想法上不断迭代，并在脑海中创建第一个设计。但是心情很长时间都没有来，所以有一次我发现自己被迫独自驾车回家两个小时。我从未试过这种方法在开车时，但我设法进入了那种状态，并想出了第一个设计。我还决定尽快原型化它。

计划是首先创建任何糟糕的设计，打印它，测试它，修改它，再次打印它。这有点令人惊讶，有趣但也很耗费精力。一开始我简单地从绘图机的整个机械配置开始。在我开始之前的主要想法。

+   *我会将步进电机放在自由移动的手臂上，橡胶轮子接触瓶子顶部，并且由于重量，它会在瓶子上施加压力以确保接触，并在运行时旋转瓶子。*

+   *笔将水平放置*

+   *将有一个相当复杂的转盘机制用于更换笔*

+   *瓶子将在底部的轴承上滚动，设计方式可以容纳不同直径的瓶子*

+   *我会尽可能地使最终产品简单/便宜。尽量使用3D打印和尽可能少的机械部件类型。我决定使用**M4x10**圆柱头作为标准紧固件。*

**剧透**：所有/大部分事情都被证明是错误的。

把第一批零件组装在一起后，问题开始显现出来。将电机放在顶部并不起作用，它试图推动瓶子但并非每次都能使其滚动。我决定很快放弃多工具，因为很快就明显地变成了一个巨大的复杂化，这应该被推迟到未来作为“**有点好**”。

最初的原型只是糟糕透顶，丑陋得不堪一击。我通常每天设计并打印大约4次零件的修订版。但我学到了很多。我使用了一些手头已有的部件，如线性台等，所以这定义了设计的某些方面。

对于电子设备，我决定使用最便宜的东西。Arduino CNC shield 和 GRBL，虽然不是很好，但很便宜，应该足够。我花了相当多的时间来调试限位开关和电机之间的干扰问题。我认为这是使用廉价电子设备的代价。

在图片上，已经有了稍微先进的版本，我开始在底部测试电动滚筒。但这也效果不佳。其中一个问题是，由于瓶子不完美，你可能在旋转前后会停在不同的位置。另一个问题是，你需要在笔的自由运动中保持跟随瓶子表面的能力。所以所有的指向都是要使笔垂直。事后看来，这显得如此明显。

我尝试使用打印弹簧作为笔座，但效果不佳，所以最终选择了我手头有的线性导轨。我本想只用重力推动笔，但最终还是用了弹簧。

我仍然在使用底部滚筒旋转瓶子时遇到问题。我希望已经有一些工业应用，需要围绕其轴旋转的应用（车床）。

*是的，我是个笨蛋...*

是的，使用轴向直驱解决了许多问题，并且更简单。一个缺点是电机的分辨率较低，因为没有齿轮或导丝使其步进更细。但这不应该是问题。

几次迭代之后锁定机制变得简单且大多数零件是打印出来的。

之后我确实拥有了一个体面的数控瓶装绘图机。**但** 它仍然基于我偶然找到的专有组件（电机、平台、导轨），所以如果我想再建一个，就不行了。

## 软件

一个常见的问题来自人们。我实际上如何准备在圆形表面上绘制数据，有趣的是，你不需要做任何事情。整台机器都像瓶子是平面表面一样旋转。因此，任何能生成2D绘图数据的软件都应该可以使用。

常见的软件工作流程是，你使用矢量图像，最好是**SVG**。然后将其转换为**G-Code**，然后通过**USB**从PC发送到**GRBL**电机驱动器。我在找到一些满意的东西方面遇到了问题，所以最后我决定自己在我的切片器/CAM上工作。这是一个简单的网页，您可以上传SVG文件，经过一些处理后得到G-Code文件。堆栈是纯JS，带有一些JQ。仍然需要改进，但它确实有效，而且在上面工作也很有趣。

## 计划

仍然存在一些关于X轴的问题，因为电机表现不佳，我认为这是便宜的驱动器和主板造成的。因此，我想测试一些新的电子设备，它们会更好，但价格并不会更贵。

还有一些设计优化的空间，可以移除一些部件，使其更便宜和简化。

我最初是在**Inventor**中设计的，但后来改用**OnShape**，因为它（至少目前）是免费的。因此，我想重新设计所有的组件在**OnShape**中，这样以后进行任何更改会更容易。

我还想开源整个设计，并开始为有兴趣的人提供套件。

## 结论

我很高兴我设法测试了快速迭代和更受控制的构建方式。再次看到最终的机器，一切都显得非常明显。这很烦人，因为我花了相当多的时间才到这一步。但这也表明，这可能是正确的解决方案。我还能够为所有组件的价格（不包括印刷件）找到大约**150美元**，这让我感到很满意。
