# Headline driven development - Slava Akhmechet

> 来源：[https://www.spakhm.com/headline-development](https://www.spakhm.com/headline-development)

# 头条驱动的开发

这里有一个有效的软件项目交付流程。首先，将项目分解成一系列的头条。然后选择一个激进的日期来发布第一个头条，并拼命努力达到这个日期。所有人只能同时工作在一个头条上——即将到来的头条。忽略其他一切。不要做任何不帮助你发布头条的事情。一旦头条发布了，切换到流中的下一个头条并重复。就是这样，你可以解雇你的敏捷顾问。

头条是一个非常简短的句子，只包含最高级的信息，剔除了所有其他信息。想象一下，你在街上偶然遇到几个月未见的人，他们问你最近在忙什么。在这种情况下，哪些回答会起作用？“我在东南亚徒步旅行”，“我换了工作”，“我结婚了”。软件发布的头条也是如此运作。“现在你可以通过API租用VM”，“我们推出了FSD自动驾驶”，“财政部在印度可用”。

基于头条驱动的开发有三个很好的原因。

首先，头条是人类处理变化的方式。如果你发现你的用户困惑，你的老板沮丧，你的投资者焦虑，你的同事漠不关心——这些问题在你围绕头条组织沟通时会消失。但这并不适用于事后添加。通过头条进行沟通，但以其他方式开发就像过着双重生活一样。这会变得非常混乱。所以，要通过头条进行沟通，你必须也要以头条方式进行开发。

其次，这使得无情地优先级排序变得容易。如果你可以可靠地发布一个头条而不需要某些内容，就删掉那些内容。例如，假设你正在准备你的东南亚徒步旅行的头条，计划访问六个国家。你能否诚实地告诉朋友们“我在东南亚徒步旅行”只访问了五个国家而不是六个？显然可以。所以一个国家被削减了。那四个国家呢？反复进行这个练习，在头条的可信度受到威胁之前停止。你希望做尽可能少的工作，同时保证头条仍然可信。

第三，截止日期效应是真实存在的。大学里的大部分工作都是在项目截止前的午夜时分完成的。行业也不例外。因此，模拟课堂作业原来是一种非常有效的快速交付方式。你需要一小块离散的工作，一个任意的截止日期，和一个二进制的结果。你通过头条来实现这一切——一个头条要么在给定的时间戳前发布，要么没有。
