# 为什么你找不到技术联合创始人

> 来源：[https://www.breakneck.dev/blog/no-tech-cofounder](https://www.breakneck.dev/blog/no-tech-cofounder)

你可能不会通过使用其中任何一个合伙人匹配工具在线找到技术联合创始人。

你可能也不想要。

我尝试过普通的“股权合作创始人”模式。我多年来一直作为自由职业者和承包商工作。

目前我正专注于打造MVP。在与股权-聘用双方创始人交谈后，我有一些想法。

这仅仅是从寻找一个你**在网上不认识**的人，**唯一目的**是**创办一家公司**的视角来看的。

这不是在谈论与你在工作中的朋友乔伊一起创业，你们已经一起工作了3年。

# [](#requirements-for-a-tech-co-founder)技术联合创始人的要求

最大的原因之一是非技术创始人期望的高标准。

你正在寻找一个

+   拥有高水平的IC（个人贡献者）能力。

    *后端、前端、托管、人工智能，可能还有设计。*

+   拥有高水平的领导能力。

    *你的想法肯定会大受欢迎，所以你不仅需要一个码农，还需要一个能够领导和激励5-20个开发团队的人。*

+   分享你的愿景/对该领域充满热情的人。

    *技术联合创始人确实对园艺、开发、人工智能、领导能力、创业精神、建筑工地安全和狗的社交媒体有兴趣。*

+   在0-2个月内可用。

+   完全免费工作。

+   拥有相同的氛围和年龄段。

    *创始合作就像婚姻一样，对吧？你们也必须个人来一致。两次会议后。*

+   至少要对你的想法保持承诺几个月。

    *可能是几年。两次会议后。*

这绝对是一个荒谬的要求清单。那种人不存在。如果存在的话，他们为什么要和你一起工作？

那样的人有其他选择。那样的人并不立即可用。那样的人不会在两次会议后就赌上他们职业生涯的三个月时间。

即使提供每月2000欧元税前的“潜在的EXISTS津贴”，你也不会吸引到任何具备这些能力一小部分的人。

“即时可用性”和“免费”部分是疯狂的乘数器。

有很多具有高水平IC能力和足够领导能力的人，但他们已经有工作，不会等着你。

# [](#opportunity-cost)机会成本

令人惊讶的是，我看到很多创始人希望运营业务方面的事务，却没有考虑这些问题。

假设开发你的MVP需要花费10,000欧元。那是一大笔钱，所以你在寻找一个技术联合创始人。

你会给他x%的公司股份，然后他开始为其建设两个月。

技术联合创始人刚刚支付了10,000欧元。在机会成本上。

*假设他可以留在工作中，每月赚取 5.000€*。

如果你考虑一下，这是一种奇怪的安排。这是你的想法。你是企业人士。

为什么你可以接受别人在你的项目上冒 10.000€ 的风险，但不接受自己的风险呢？

你有愿景，对吧？你有激情，对吧？

# [](#risk)风险

就像机会成本的概念一样，我看到很多企业创始人都非常风险厌恶。

当与客户讨论潜在的MVP开发时，创始人对“股权 vs 雇佣”的回答将归结为“嗯...可能不会很成功，然后我损失了 10.000€”。

他们很少会明确说出来，但这就是贯穿他们所有思考的东西。

是的。是的，可能不会成功。欢迎来到商界，欢迎开始创业。

存在风险，不要把它们全部推给你的技术联合创始人。

你应该意识到这些风险，这些隐藏成本，并可能承担至少50%。

# [](#your-level-as-the-business-person)你作为企业人士的水平

非技术创始人通常从事产品、市场营销或销售，或者在自己行业中有深厚的专业知识，并发现了一个问题。

你也是创业初期的**企业和财务方面**。寻找高水平的技术人员，就像需求部分所讨论的那样。

如果你已经 25 岁以上，想要创业，却没有 10.000€ 的资金，我对你的能力表示怀疑。

你是企业中的资金部分。10.000€ 是一大笔钱，但如果你想要开创一家公司，并且你认真对待它，就像你向外界展示的那样：

**这**应该是你要解决的第一个问题。

通过与 20 名开发人员交谈，看看谁与你合得来，找不到技术联合创始人。

# [](#reasons-you-might-want-to-pay-for-a-developer)你可能想为开发人员付费

### [](#you-find-someone-way-quicker)你可以更快地找到合适的人选。

回到机会成本。如果你额外花了 6 个月的时间寻找联合创始人，那可能是额外收入的 6 个月。

### [](#you-might-find-a-co-founder)你可能会找到一个联合创始人。

他在需求部分勾选了很多选项。

你付钱给某人，与他们合作，看看你们的合作情况如何。如果效果不错，你可以考虑不同的安排。现在你有更多信息来做出决定。

### [](#you-dont-need-to-make-sure-they-buy-into-your-vision)你不需要确保他们对你的愿景感兴趣

你会惊讶地发现，有些人在建立他们并不关心的软件方面做得很好，为他们并不喜欢的客户工作。你实际上有充裕的时间来确认你们是否合得来。

### [](#you-get-far-less-scrutiny)你会受到远少于技术人员的审查。

如果你是客户，我不需要质疑你的想法或你的能力。

**你**做了所有这些工作，你不需要说服开发人员这是个好主意，你擅长自己的工作。

### [](#you-take-on-100-of-the-risk-for-100-of-the-reward)你承担全部风险，获得全部回报。

有点不言自明。为了一个初步开发的3-6-12个月，将你企业的50%可能大大低估了。你怎么知道？何不在有更多信息时再做决定？

### [](#you-get-a-much-higher-commitment-level)你获得了更高的承诺水平。

这就像销售技巧，你给别人一些东西，现在他们觉得欠你一样。

他们确实欠你一定的结果或者时间量，没错。

但我更多地谈论的是人们在付费和无偿工作时对工作的认真程度的感觉。

我只能谈论我自己，但我注意到当我得到报酬时，我的思维方式不同。突然间，我专注于质量并深入研究，而不是因为有人觉得有权要求我为可能最终有效果的更多工作而感到微妙地恼火。

特别是在开始阶段，这种感觉是潜在的。

即使在独自为自己工作时，即使它正在赚钱，我对它的认真程度要低得多。

有些事情是关于获得报酬并对另一个人负责的，这种感觉是完全不同的。

### [](#you-put-off-decisions-to-a-later-point-with-way-more-information-available)你推迟决策，等待更多的信息。

在两次Zoom会议中评估一个陌生人的能力试图笑话。他们的技术能力、个性、承诺水平。

讨论公平的百分比、公平的工时，以及如何在未来的公司不同阶段分配责任，完全是一次瞎猜。

# [](#i-have-absolutely-no-money-now-what)我完全没有钱了，现在该怎么办？

第一步是意识到你所处的情况。如果你是21岁，学习经济学，只是想尝试一些疯狂的事情，那太棒了。我不指望你有钱支付任何人。

你应该相应地行事。让它变得有趣。心存感激。意识到你实际上是在请求别人的空闲时间。

不要假装成熟的商人，仅仅因为你和投资者有个会议或者你有一个每月由国家资助的1500欧元的津贴。

与水平相近的人合作。

软件开发是一个奇怪的领域，21岁的大学生可以自学建立一个完全功能的产品。意识到你在赌博，寻找这样的人。

开发者们疯狂。人们免费为开源产品工作多年。人们在一天写代码的工作后，还要抽出时间写他们的副业项目。

尝试让某人度过一个具有挑战性的时期。

从你的个人资料中删除掉 ChatGPT 生成的技术联合创始人的需求列表。

这太荒谬了。我知道你甚至不知道那意味着什么。

为什么要试图让它看起来像任何其他普通的职位发布，只是没有薪酬部分？

设定一个目标，存下1万欧元，然后雇人。你可以从那些以此为生的人身上学到很多。

然后在你的下一个项目中，你突然有了开发工作流程、流水线、生产和暂存环境的经验，听起来更有吸引力。

冒险尝试许多人，快速地。忽略你那荒谬的需求清单。试试人，看看会发生什么。

# [](#other-misconceptions)其他误解

### [](#the-person-who-built-it-initially-needs-to-stick-around)最初建造它的人需要留下来

当然，这很好，而且纯粹优于其他选择，但是它被夸大了。

在现实世界中，开发者几乎从不参与项目的起始阶段。即使参与了，也可能在1-2年后离开。

在现有项目中工作也可能更容易，因为有固定的做事方式。不需要做决策，大多数问题都已解决。

你不会遇到“我们该用什么框架？”，“我们该用什么UI库？”，“我们如何在客户端验证？”，“我们如何推向生产环境？”这样的问题。

这些都已经被找出并决定了。你可以在代码库中找到几乎所有例子并继续扩展项目。

### [](#loyalty-differences-between-co-founders-freelancers-and-employees)联合创始人、自由职业者和员工之间的忠诚度差异

很多创始人认为只有股权才能使人忠诚。或许是股权和对愿景的认同。

全球90%的软件项目由员工管理。通常甚至不是公司的直接员工，而是产品公司雇佣的开发代理机构的员工。

无论好坏，人们都对自己的工作负责并承担责任。

我们都希望我们每周工作40小时的工作很重要。

这就是为什么每个开发者的帖子都以“我正在处理一个关键项目”开始。

每位设计师都认为可访问性很重要。

经营自行车修理店的人认为你两个月前买的500欧元的自行车是无用的垃圾。

这只是人类的本性。

作为一个创业理念的人，你认为对你的理念进行认同很重要。

我也是支持激励措施对齐的粉丝。我认为这只是合理的，所有员工的标准情况，他们的薪水与结果完全脱钩并不理想。

总的来说，我看到辞职的人比辞职的股权型创业公司的人少得多。

### [](#realize-your-bias)认识到你的偏见

你是商业人士。企业家。创意人。

自然而然地，你认为每个人都像你一样对此极度重视很重要。

但就像你不认为设计那么重要一样，其他人也不认为创意那么重要。

如果你已经有1-2位商业合伙人，不要寻找第三位既有技术又有商业能力的合伙人。

让人们专门化并随时间适应。

### [](#passion)激情

我看到创始人们经常强调这一点。

对我来说，这显示了成年人对激情如何运作的天真和根本性的误解。

激情是随着时间而发展的。

你**刚刚**告诉我关于组织变革管理，现在你期望我对此感兴趣？

哦，现在你希望我对太阳能技术感兴趣？

哦，现在你希望我对医疗保险感兴趣？

哦，现在你希望我对退休院的餐饮计划感兴趣？

哦，现在你希望我对推荐正确的电视节目感兴趣？

哦，现在你希望我对让人们成为素食主义者感兴趣？

开发人员经常在你从未听说过的领域工作。特别是在第一天，激情并非必需。

开发人员热衷于用代码解决问题。

开发人员热衷于构建酷炫的东西。

开发人员热衷于键盘、编辑器和快捷方式。

你需要你的会计师有多么热衷？

问问自己，你对**软件**有多热衷？你现在想要构建一个吗？

# [](#final-thoughts)最后的思考

减少对技术联合创始人的需求清单。意识到，直到有付费客户之前，你实际上是在请求别人的帮助。

考虑**成本**、**机会成本**以及支付和不支付某人来推动你的想法的**风险**。

不要期望从别人那里得到与**你的**热情水平和对**你的**想法的承诺水平相同的东西。

尤其是不是在2次会议之后。
