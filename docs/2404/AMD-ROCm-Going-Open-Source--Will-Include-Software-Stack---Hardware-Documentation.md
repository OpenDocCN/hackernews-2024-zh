# AMD ROCm 将开源：将包括软件堆栈和硬件文档

> 来源：[https://wccftech.com/amd-rocm-going-open-source-will-include-software-stack-hardware-documentation/](https://wccftech.com/amd-rocm-going-open-source-will-include-software-stack-hardware-documentation/)

AMD 计划在未来的更新中开放其 ROCm 软件堆栈和硬件文档的部分内容，旨在为消费者精细调整其软件和 AI 生态系统。

## AMD 表示，将根据社区的反馈开放其 ROCm 软件堆栈和硬件文档的部分内容。

AMD 最近因其 ROCm 软件和 AI 支持而陷入热水中。像 [George Hotz](https://twitter.com/__tinygrad__/status/1770155923211997403?ref_src=twsrc%5Etfw%7Ctwcamp%5Etweetembed%7Ctwterm%5E1770155923211997403%7Ctwgr%5Eb225cbf7218d4c79f18ff26dfe9c88ccc74c241a%7Ctwcon%5Es1_&ref_url=https%3A%2F%2Fwccftech.com%2Ftinycorp-ditches-amd-tinybox-ai-package-opt-for-nvidia-or-intel-options%2F) 的 TinyGrad 等公司批评 AMD 的驱动支持不稳定，导致使用 Radeon RX 显卡的体验非常不稳定。提出的解决方案之一是使 Radeon 软件和 ROCm 开源，以便社区能够提供驱动程序修复和各种优化。

好消息是，[AMD 已经听取了反馈意见](https://twitter.com/amdradeon/status/1775261152987271614?ref_src=twsrc%5Etfw%7Ctwcamp%5Etweetembed%7Ctwterm%5E1775261152987271614%7Ctwgr%5E8f021c3e633e270f41bec7a94bc9403646d6e905%7Ctwcon%5Es1_&ref_url=https%3A%2F%2Fvideocardz.com%2Fnewz%2Famd-to-open-source-more-of-radeon-software-provide-more-hardware-documentation)，现在打算开放其软件堆栈和硬件文档的额外部分。此外，AMD 还创建了一个名为 "[fuzzyHSA](https://github.com/nod-ai/fuzzyHSA/issues)" 的跟踪器，用于收集反馈并提供实时更新。对 TinyCorp 的回应是，AMD 正专注于提供软件和固件更新，以解决报告的问题。他们的软件团队目前正在进行相关工作，并将尽快在 GitHub 上报告最新状态。

到目前为止，AMD 在[将 ROCm 6.1 支持带到其消费级硬件，例如 Radeon RX 7900 XTX、Radeon RX 7900 XT、Radeon RX 7900 GRE、Radeon Pro W7900 和 Radeon Pro W7800](https://wccftech.com/amd-preps-for-rocm-6-1-release-coming-enhanced-support-features/)方面做得相当不错。这些加上一系列的 AI 优化，已经构建了强大的软件堆栈，而且开源的性质可能会帮助他们进一步推动事务。

公司正在与[NVIDIA的CUDA](https://wccftech.com/nvidia-halts-use-of-cuda-on-other-platforms-lists-new-warning-in-the-eula/)和[Intel的oneAPI](https://wccftech.com/qualcomm-intel-google-join-hands-to-come-nvidia-plans-on-dethroning-cuda-oneapi/)等竞争对手竞争，在AI领域，前者目前是无可争议的领导者，而后者则由包括芯片巨头在内的多家公司共同开发，旨在应对CUDA。因此，当AMD的ROCm在开源启动后将如何发展，将会非常有趣。
