# Arc浏览器登陆Windows，挑战Chrome和Edge - The Verge

> 来源：[https://www.theverge.com/2024/4/30/24144183/arc-browser-windows-launch-features-availability](https://www.theverge.com/2024/4/30/24144183/arc-browser-windows-launch-features-availability)

几年来一直让macOS和iOS用户印象深刻的出色Arc浏览器今天终于[登陆Windows平台](https://arc.net/)了。Arc旨在从根本上改变您使用浏览器的方式，包括可折叠的侧边栏，将垂直标签和书签结合为应用切换器式体验，用于导航的命令栏以及帮助您浏览Web的实用工具。

背后是Arc的公司，名为浏览器公司，正打赌他们的浏览器足够与Chrome和Edge不同以吸引Windows用户。“Arc真的只是让人感到平静并保持有序，”浏览器公司的联合创始人Hursh Agrawal在接受*The Verge*采访时说。“它帮助您更好地处理一天中的任务。”

作为一个Chrome的忠实用户，使用Arc是一个令人震惊但收获颇丰的体验。一旦你习惯了Arc的使用方式，用侧边栏而不是标签行或者只需移除所有浏览器杂乱并专注于单个网站的能力，感觉像是对我们今天使用Web的一种清新的方法。Arc还是基于Chromium构建的，因此网站可以正常工作，并且你可以使用你的扩展程序。

*你可以在Arc中简化整个浏览体验。*

Image: 浏览器公司

Arc for Windows也是浏览器公司所谓“互联网操作系统”的更大愿景的重要一步 — 这个概念认为Arc是一个超越仅仅是浏览器的开放Web平台。Arc for Windows对于这一愿景来说是一个重要的里程碑，其中一个原因是浏览器公司还帮助将由苹果创建的用于构建iOS和Mac应用的编程语言Swift带到了Windows平台。

Arc在Windows上使用Swift构建，浏览器公司的软件工程师们在很大程度上开源了所有用于构建Swift Windows应用程序的基础设施。Saleem Abdulrasool是Swift核心团队成员，也是浏览器公司的软件工程师，他花了超过六年时间将Swift带到Windows平台。“我对此非常兴奋，”Abdulrasool在接受*The Verge*采访时说。“终于看到长时间努力的成果真的很不容易。这真的是一次爱的付出。”

由于 Swift，在 Windows 上发布的 Arc 可能会鼓励其他开发者走类似的道路，特别是在白宫敦促软件开发者使用内存安全编程语言如 Swift 替代用于构建 Chrome 和 Edge 的 C++ 语言之后。“我们实际上也邀请其他人参与，”Abdulrasool 表示。“虽然我们领导着这一进程，但我们非常乐意与他人一同前行。”

*Arc 的画中画模式。*

图片：The Browser Company

尽管 Arc 有着令人印象深刻的基础，它仍然需要足够好，以应对 Chrome 的竞争。毕竟，它被市场定位为“你一直在等待的 Chrome 替代品”，因此对于这个 Windows 版本来说，压力很大。The Browser Company 还将不得不应对微软 [强硬的尝试](/23935029/microsoft-edge-forced-windows-10-google-chrome-fight) 将 Windows 用户转向 Edge，以及颠覆 Chrome 的努力，这是任何浏览器竞争者在过去几十年中都没有面对过的挑战。

对于 Darin Fisher 来说，这是他长期以来建立颠覆现状的网络浏览器的最新挑战。Fisher 在 2022 年末加入了 [The Browser Company](/2022/10/31/23428862/arc-browser-web-company-darin-fisher)。他的职业生涯始于早期的 Netscape，后来帮助将 Navigator 转变为 Firefox，并花了 16 年时间建立 Chrome 和 ChromeOS。

Fisher 认为 Arc 在市场上存在一个空白，而竞争对手的浏览器已经停滞不前。“我在谷歌看到了这种转变，人们从普遍使用 Internet Explorer 和 Firefox 转向突然间使用 Chrome，”Fisher 在接受 *The Verge* 采访时说道。Internet Explorer 已经停滞不前，而 Chrome 在一个更好的时机打击了市场。这正是 The Browser Company 希望复制的模式。

它还试图通过新工具做到这一点。“我们如何应对生活的复杂性？我们需要帮助我们在哪里的工具，这就是 Arc 的全部意义，”Darin 说道。在 macOS 版本中，您可以自定义网页以更改它们的色彩调色板，尝试 [新的 AI 功能](/2024/2/1/24058013/arc-browser-smart-folders-browse-for-me-ai)，或创建可分享的网站。目前，这些受欢迎的功能并不是都已经在 Windows 版本中实现，但像侧边栏、Spaces、配置文件、分屏视图和画中画这些基本功能现在已经都可以供 Windows 用户使用了。

幸运的是，Arc 每周都会继续发展，这对于 Windows 版本的功能特性逐步与 macOS 版本达到平衡至关重要。“我觉得对于一些人来说，这可能有些可怕，但它每周都在改进，”Agrawal 表示。“人们说这就像每个星期四收到圣诞礼物一样。”

目标是构建 Windows 版本，并移除一些不常用的 macOS 特性，直到它们作为一个单一应用达到功能平衡。“Arc 在 Windows 上的体验很好，但还不够完美，我们每周发布新版本，以实现令人惊叹的效果，”Agrawal 承认道。

最重要的是，Arc 在 Windows 上并不像 macOS 应用那样。浏览器公司使用了微软的 WinUI 框架，遵循了 Windows 11 的 Fluent 设计语言，甚至可以调整窗口背景到 Mica，以匹配 Arc 的背景到你的桌面，或者使用 Acrylic 实现透明模糊效果。

最初，Arc for Windows 仅支持 Windows 11，计划不久推出 Windows 10 版本。尽管 Google 最近为 Windows 推出了 [Chrome 的本机 ARM64 版本](/2024/3/26/24112273/chrome-native-windows-on-arm-qualcomm-snapdragon)，但 Arc 将不会在即将到来的夏季使用 ARM 处理器的 Windows 笔记本上本地提供支持。

“我们已经将 Swift 移植到 Windows ARM64 平台上运行，这意味着我们可以考虑迁移到这个平台，” Abdulrasool 解释道。“我们正在调查这个问题，但目前并非当务之急，因为很多人还没有这种硬件的访问权限。”

Arc for Windows 发布后还将与 Arc Search 同步，这是之前只能与 Arc for macOS 一起使用的 iOS 应用。浏览器公司还在开发 Arc Search 的 Android 版本，这将显著促进 Arc 在多平台上的使用。

Arc for Windows 现在可以立即下载，并且预计会看到每周的更新，因为浏览器公司致力于实现 Windows 和 macOS 之间的功能平衡。
