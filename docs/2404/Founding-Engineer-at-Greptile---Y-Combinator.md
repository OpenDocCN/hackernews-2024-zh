# Greptile的创始工程师 | Y Combinator

> 来源：[https://www.ycombinator.com/companies/greptile/jobs/4MY7eZL-founding-engineer](https://www.ycombinator.com/companies/greptile/jobs/4MY7eZL-founding-engineer)

嗨！我们正在寻找一名创始工程师加入我们的Greptile团队。

**关于Greptile：**

Greptile正在构建一个AI专家，适用于任何代码库，作为API，因此团队可以轻松构建强大的内部AI开发工具。

开发人员可以将Greptile指向任何GitHub/Gitlab仓库，Greptile对其进行索引（新提交时自动更新），并允许您以自然语言查询。 Greptile使用代码库优化版的RAG，模拟人类开发人员如何浏览代码库，其性能比朴素的RAG高得多。

您可以在此处免费尝试我们的聊天UI，包括150多个开源软件库[此处](https://learnthisrepo.com)。

一些内部工具团队可以/已经构建的示例：

1.  **工单增强器：** 添加代码库上下文到仓库中，包括要查看的特定文件路径。

1.  **自动文档更新器：** 每次推送新代码时编写PR以更新文档。

1.  **PR审核机器人：** 审查PR并留下带有完整代码库上下文的评论。

[查看更多。](https://docs.greptile.com/examples)

我们是一个由3位技术联合创始人组成的团队，驻地在旧金山。

**要求：**

+   强大的编程技能，特别是在TS/JS方面。

+   计算机科学学士/硕士学位

+   在美国有工作授权

+   有能力并愿意搬到旧金山亲自工作。

**首选资格：**

+   在开源社区做出贡献的历史。

+   对开发者工具充满热情。

+   有能力相对独立地工作。

+   强大的副业项目，最好使用LLMs。

+   一些DevOps经验，理想情况下使用AWS。

+   一些LLM-Ops经验，与VectorDBs/LLMs合作。

**技术栈：**

我们的大部分代码库都是用Typescript编写的。我们的后端基础设施托管在AWS上，使用ECS容器和Lambda函数的组合 + Postgres和DynamoDB作为数据库。前端使用NextJS。
