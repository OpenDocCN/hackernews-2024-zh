# 开源社区联合构建符合 CRA 的网络安全流程 - Apache软件基金会博客

> 来源：[https://news.apache.org/foundation/entry/open-source-community-unites-to-build-cra-compliant-cybersecurity-processes](https://news.apache.org/foundation/entry/open-source-community-unites-to-build-cra-compliant-cybersecurity-processes)

作者：Dirk-Willem van Gulik，公共事务副总裁

全球政府对网络安全的重视程度日益增加。欧盟的《网络韧性法案》（CRA）引入了关于如何开发、测试、审计和支持软件的规则，以确保更安全的软件。由于开源软件支撑着当今全球的数字基础设施，这对开源软件生态系统中的许多参与者产生了深远影响。

Apache软件基金会、Blender基金会、OpenSSL软件基金会、[PHP基金会](https://thephp.foundation/blog/2024/04/02/open-source-community-cra-compliance-initiative/)、[Python软件基金会](https://pyfound.blogspot.com/2024/04/new-open-initiative-for-cybersecurity.html)、Rust基金会和Eclipse基金会共同宣布我们打算合作，制定基于现有开源最佳实践的安全软件开发通用规范。这个工作组将解决开源生态系统中多方面的网络安全挑战，并展示我们与 CRA 合作和实施的承诺。

最初，该组织的工作将在列举各开源基金会现有的安全政策和程序以及类似的描述最佳实践的文档方面展开。多年来，这些基金会和社区一直在创建并维护安全软件开发流程的行业最佳实践。以这些最佳实践为起点，我们旨在加速形成符合监管合规要求的协调的网络安全流程，并为与广大开源社区进行技术讨论提供中立的环境。

各基金会、供应商、社区等的中立性对此努力至关重要。新工作组将由总部位于布鲁塞尔的Eclipse基金会AISBL主办，遵循Eclipse基金会规范过程的指导。工作组的治理将遵循Eclipse基金会的常规会员主导模式，但将通过开源社区的明确代表确保决策的多样性和平衡。成果将包括一个或多个过程规范，以自由规范版权许可证和免费专利许可证发布。

阅读[Eclipse的博客](https://eclipse-foundation.blog/2024/04/02/open-source-community-cra-compliance/)，了解开源社区的具体影响，包括引入开源管理者，以及[我的上一篇博客](https://news.apache.org/foundation/entry/update-on-eu-software-regulation-lots-of-improvements-good-news)，如果你希望深入了解CRA的内部工作机制。

这是开源社区共同努力确保更安全未来的关键时刻。我们欢迎所有代码托管的开源基金会、中小企业、行业参与者和研究人员加入这一合作。要获取此倡议的最新信息，请[注册工作组的邮件列表](https://outreach.eclipse.foundation/open-source-cybersecurity-specifications)。
