# 救命！我的鼠标爬上墙，而且它不能正常工作 • The Register

> 来源：[https://www.theregister.com/2024/04/26/on_call/](https://www.theregister.com/2024/04/26/on_call/)

当历史的另一个星期流逝时，是时候再次为*The Register*提供On Call的新篇章了——我们每周读者贡献的技术支持痛苦和胜利的故事。

本周，让我们认识一个名为“Barry”的读者，他分享了一个关于上世纪90年代初他作为IT学生的故事。有一个夏天，Barry在一家建筑公司担任技术实习生。

“那时还是5 1/4英寸软盘的时代，而且鼠标仍然是一种新奇事物，”他告诉On Call。

这样一个新奇的事情，Barry很快发现自己接到了一个架构师的电话，抱怨他的鼠标无法到达屏幕的上半部分。鼠标指针会达到屏幕的一半，然后停止。

当Barry走向用户时，他想象也许是鼠标球变脏了——这在老式鼠标使用滚轮检测运动时是一个常见问题，滚轮擦过内部的球体。当用户移动鼠标时，球在他们的桌面上拖动并开始旋转，推动滚轮，最终导致指针移动。

这些鼠标球有时会粘上绒毛并卡住鼠标内部的滚轮，因此Barry的假设适合那个时代。

在这种情况下也是错误的。鼠标球没问题。问题在于架构师并不知道可以把鼠标拿起来重新放置而不会导致指针从当前位置移动。

而架构师将鼠标移动到它能到达的最远处，但这并不远，因为当时的鼠标通过线缆连接到个人电脑。

“他把鼠标在他的桌子上越来越远地移动到一个点，到达一个他根本无法再把手伸得更高的地步，”Barry告诉On Call。

Barry从未预料过，他站起来推动鼠标往墙上走。

“我必须承认，在轻轻地和温和地解释他的问题的激进解决方案之前，我眨了几次眼睛并停顿了一下，”Barry告诉On Call。

架构师出奇地接受了这个消息。

“幸运的是，他没有变得防御，而是默默接受了这个消息，并继续了他的一天，”Barry告诉On Call。

这次经历至今在他脑海中仍然清晰，因为它教会了一个重要的教训：“你绝不能对用户有任何假设，”Barry写道。
