# tea.xyz再次引发开源软件垃圾邮件问题

> 来源：[https://www.web3isgoinggreat.com/?id=teaxyz-causes-open-source-software-spam-problems-again](https://www.web3isgoinggreat.com/?id=teaxyz-causes-open-source-software-spam-problems-again)

tea.xyz协议首次出现在*Web3 is Going Just Great*上是在二月末，当时他们计划奖励[开源软件](https://en.wikipedia.org/wiki/Open-source_software)的贡献者，结果是加密货币爱好者们无意参与开源软件贡献，却开了无数的PR请求，试图获取著名开源项目的所有权。这种垃圾邮件对这些项目造成了破坏，通常由志愿维护者需要弄清楚发生了什么，然后试图停止这些垃圾PR。

[tea.xyz](https://www.web3isgoinggreat.com/?id=teaxyz-causes-open-source-software-spam-problems-again)的创始人Max Howell（也是Homebrew的创始人，尽管他现在已经不再参与）显得很抱歉，并承诺修改协议以阻止这种垃圾行为。

现在，由于这个途径被剥夺，人们只是在创建大量的空软件包浪潮，里面什么都没有，只有一个“teafile”和他们的加密钱包地址用于奖励，并将它们提交给像[NPM](https://en.wikipedia.org/wiki/Npm)和[RubyGems](https://en.wikipedia.org/wiki/RubyGems)这样的软件包管理器。

这次垃圾邮件促使[RubyGems](https://en.wikipedia.org/wiki/RubyGems)发布了一篇博客文章，他们在文章中写道，他们不得不花时间加强软件包发布的限制，并“确保[账户]不会进一步干扰社区。”

Phylum的安全研究人员还详细描述了该协议对JavaScript世界的影响，NPM上发布的软件包数量甚至高达以前每日平均数的7倍。他们写道：“几个月来自动化持续的大规模垃圾邮件攻击是罕见的，它除了给生态系统本身带来重大压力外，对于真正的用户和开源安全研究人员来说也会导致性能下降。”
