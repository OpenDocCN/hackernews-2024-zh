# [Michał Sapka's website] BSD 可能不适合你

> 来源：[https://michal.sapka.me/bsd/why-not-bsd/](https://michal.sapka.me/bsd/why-not-bsd/)

更换 GNU/Linux 发行版可以随心所欲，因为在所有这些底层，你都拥有相同的基本操作系统。使用 BSD 系统就不同了。人们应该试着了解其缺点，以免浪费接下来的 20 年去探索一个根本不适合的操作系统。

## 硬件

所有 BSD 系统都比 GNU/Linux 要少得多的用户群^(，这也带来了最紧迫的缺点。硬件厂商的支持，至少是有问题的。你会遇到新发布的硬件组件问题；你的 WiFi 卡很可能无法正常工作。你的显卡可能有驱动程序，但性能可能会更慢。必须深入检查手头的硬件，并确保它被所选操作系统支持。例如，我的笔记本电脑有一张 NVIDIA 显卡硬接到 HDMI 输出，因此如果我想使用外部显示器，我需要使用这张显卡。在 FreeBSD 上这是可以工作的，但在 OpenBSD 上没有支持。另一个例子是我不得不物理替换的 Intel 无线网卡^(以避免持续的内核恐慌)。无需多言，电源管理也是一个问题，但在 GNU/Linux 上也是如此。))

供应商的不足支持由志愿者处理，他们试图逆向工程硬件。又来了另一个问题 - 从GNU/Linux内核反向移植任何东西**非常**困难。GPL和BSD许可证允许的内容之间存在巨大差异。即使这不是问题，GNU/Linux的驱动程序领域充满了闭源块。你可能会认为，如果某些东西在那里得到支持，它有美丽的开源驱动程序。这与事实完全不符。你曾经想过为什么GNU的GUIX不支持Nvidia吗？那是因为驱动程序以二进制形式提供，因此是闭源的。因此，为BSD支持所需的工作是困难和缓慢^(，^。)

## 软件可用性

但假设这在你的情况下不是问题。你被硬件支持之神眷顾了。你已经安装了系统：你的GPU正在计算像素，你的空气中充满了位和波。下一步呢？软件！又遇到另一个问题：软件支持。大多数流行的软件没有针对BSD操作系统进行编译。VS Code？Steam？Microsoft Office？Firefox？这些程序可能对你的工作或娱乐至关重要。在桌面BSD用户群体规模不够大，以至于公司不愿意与我们打交道。我们在这里并非完全束手无策，因为BSD符合POSIX标准，所以可以编译运行在GNU/Linux上的所有东西。这需要一些改动和一点运气^(但任何开源应用程序都可以运行。不幸的是，因为我们不使用GNU/Linux，使软件在那里运行快的所有黑客可能现在在这里不起作用，甚至会造成巨大问题。在FreeBSD上的Firefox是一堆层叠的补丁。我不知道支持它需要多少工作，但在“大”和“饶命”之间吧。至于那些闭源的软件？没有Office适合你。)

## 帮助

让我们再次放飞我们的想象力，假设你使用的所有软件都在那里，但出现了问题。你尝试使用Kagi进行解决，但什么也没有。GNU/Linux有大量的网站、博客和视频博客。你可能遇到的任何问题，别人都已经解决并记录了。在BSD中，你应该阅读手册。但等等，你可能会问，RTFM？这很粗鲁！然而，这是事实。因为所有的BSD都有最好的文档，所以人们会假设你首先在那里寻求帮助。这也意味着一些琐碎的问题可能永远不会成为任何博客文章的主题。不幸的是，非琐碎的问题通常也很少有记录。社区是友好的，会帮助你，但你需要做好功课。由于社区规模较小，可能需要一些时间/运气，但总会有人帮助你自己解决问题。当GNU/Linux可以被一个对操作系统内部工作一无所知的人使用时，BSD就不会那么友善了。

请注意，我像一个绅士一样，在提到Linux时总是提到GNU？嗯，BSD教会你为什么应该这样做。因为BSD和Linux使用不同的用户空间软件，它们不兼容。虽然是的，像ls、cat、awk这样的基本用法是相同的，但GNU喜欢添加大量自定义扩展。你可以假设**只有符合POSIX标准的要求**。因此，你会在网络上找到答案，但它们可能对你无效，因为它们是为GNU风格的工具编写的。不幸的是，POSIX是一个更文明时代的武器。现在的人们假设*很多*，而BSD甚至没有默认安装ZSH。你曾经使用过基本的*Shell*吗？太糟糕了，因为*Shell*是你应该在所有脚本中假设的内容。

## 变革的步伐

接下来：你喜欢称自己为*早期采纳者*吗？身处*最前沿*是你的动力？BSD 凭设计演化缓慢。如果某物管用，就别动它。这就是信条。GNU/Linux 正在快速变化 - Pipewire、Wayland、SystemD。甚至好老的*ifconfig*也被弃用了。与此同时，BSD 仍然使用几十年前的技术^(。从未有过替换的必要，所以也没人这么做。)

## 它是 Linux，对吧？

最后，准备好迎接很多异样的眼光。年轻人可能从未听说过 BSD。他们摇摆着自己的 MacBook，不知道（也不关心）它的 FreeBSD 背景。在我接触到的现实世界中，很少有人见过 BSD 系统，而且要解释它不是 Linux 也不是那么容易。

你可以把这些问题看作是试图在 1999 年用 RedHat 在装有 winmodem 的计算机上运行时遇到的问题。它不是一个会让你能专注工作的操作系统。你需要*享受*让它为你工作的过程。否则，你只会发现烦恼和迅速的操作系统更换。
