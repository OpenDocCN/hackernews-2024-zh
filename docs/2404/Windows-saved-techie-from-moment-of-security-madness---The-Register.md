# Windows拯救了技术人员，使其免于安全疯狂时刻 • The Register

> 来源：[https://www.theregister.com/2024/04/22/who_me/](https://www.theregister.com/2024/04/22/who_me/)

读者们，又是星期一了，亲爱的读者们，你知道这意味着什么：再次深入Who, Me?的忏悔，分享那些IT出错的故事，*Reg*读者们设法假装已经处理得很好。

本周，我们见到了一位我们将称之为“德克兰”的读者，他自称是“通过CAD设计机器的设计师”——但他的技术洞察力足够，使他基本上自学了如何使用软件。确实，在他90年代的第一份工作中，他被视为“技术人员”和支持的首选。

他所服务的咨询公司有一堆Windows机器，几台Unix盒子——大多数来自Sun和Silicon Graphics——还有一台数字Alpha RISC机器，运行着专门为这些盒子打造的Windows NT的特殊版本。对于那些不记得这个短命项目的人来说，它基本上是微软对于世界上有非英特尔处理器这个想法的半心半意的回应。大部分将NT移植到Alpha架构的工作由Digital完成，当这家公司被康柏收购时，这个梦想也宣告结束。

总之，长话短说，Windows NT能在Alpha架构上运行，但效果不是很好，并且几乎没有本地应用程序——几乎所有东西都在仿真中运行。然而，当时的500MHz Alpha处理器足够快，性能是足够的。这就是德克兰使用的机器。

在他的CAD工作之外，德克兰还有各种各样的职责，包括安装Windows、支持网络，尤其是尝试阻止他的同事传播病毒。

德克兰明白，一旦病毒侵袭雇主的网络，可能会导致工作陷入停顿，因为当时的防病毒软件大多在感染广泛后才更新。因此，他在指导每个人对来自不信任来源的附件或陌生文件要谨慎打开方面非常勤奋。

如今如昔，预防胜于治疗。

如今如昔，人们总是会犯错误。

一个疲惫、紧张的下午，德克兰收到了一封包含Excel电子表格的电子邮件。他认为看起来足够合法，于是双击打开了它。

他的屏幕立即被错误信息遮盖。他感到恐慌，意识到电子表格必定包含宏病毒。然后他意识到，他正打算通过做任何事情来带领公司崩溃，而这正是他警告其他人不要做的事情。哦，羞耻！耻辱！

不过，片刻之后，当他阅读错误信息时，他感到一种平静降临在他身上。

电子表格确实感染了病毒。然而，它所含的蠕虫设计用来打开收件人在Outlook中的联系人列表，并将自身发送给所有人，从而继续传播——毫无理由地继续传播。

在勒索软件出现之前，世界是多么美好啊！

这只蠕虫没有料到的是，Declan 在仿真环境中同时运行 Excel 和 Outlook，但它们之间的集成非常糟糕，导致无法正常工作。这只蠕虫一直试图创建电子邮件并发送，但每次都失败。

Declan 设法停止了自己机器上错误消息的洪流，并意识到他永远不必告诉任何人他的严重疏忽。
