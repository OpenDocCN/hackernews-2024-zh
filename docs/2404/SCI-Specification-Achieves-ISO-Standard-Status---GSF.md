# SCI 规范成为 ISO 标准 | GSF

> 来源：[https://greensoftware.foundation/articles/sci-specification-achieves-iso-standard-status/](https://greensoftware.foundation/articles/sci-specification-achieves-iso-standard-status/)

发布于 2024 年 4 月 25 日

我们为绿色技术取得了一次重大胜利：软件碳强度（SCI）规范 v1.0 现在是 ISO 标准。

随着版本 1.0 的批准和发布，ISO 证实 SCI 规范是衡量和减少软件碳足迹的可靠、公平和可比较的协议。SCI 规范的最新版本 1.1 现已发布，可以在[这里](https://sci.greensoftware.foundation/)访问。

"达到这一里程碑标志着 GSF 成员组织的集体努力，" 强调了 Sanjay Podder，负责人工智能首席技术官兼技术可持续创新主管，以及 GSF 主席。" 这一成就释放了实现 SCI 规范全行业采纳的潜力，这对于帮助组织实现真实的减排目标至关重要。"

应用 SCI 规范使用户和开发者能够根据碳使用考虑因素做出明智选择，进而推动技术行业内更环保可持续的实践采纳。

与某些框架不同，SCI 规范不将中和或抵消纳入其计算中。相反，它强调真正减少碳排放的努力。

通过采用 SCI 规范，ICT 和技术公司可以获得标准化协议，评估其软件在减少排放机会方面的表现。这一里程碑式的发展使组织能够展示其在绿色软件开发中的质量、效率和可靠性承诺。

"我们相信开源的 SCI 规范将在技术行业内引发可持续创新的新时代，" Henry Richardson，WattTime 的高级分析师和 GSF 标准工作组的联席主席补充道。" 希望这也是另一个证明开源社区集体贡献对于扩展绿色软件的重要性的指标。"

技术可持续性是共同责任，拥有正确的工具对于在规模上实现 SCI 规范至关重要。" 我们还在利用像 Impact Framework 这样的工具来轻松计算 SCI 分数，" 补充了 GSF 标准工作组和 Impact Framework 的联席主席 Navveen Balani。" 我们预见 SCI 测量将成为每个开发者工具箱的标准部分，并在未来成为常态。"

GSF 成员组织，包括 Accenture、AVEVA、CAST Software、Intesa、Microsoft、NTT DATA、UBS 等，通过各种使用案例使 SCI 规范成为可操作的。[**这些示例是公开可用的**](https://github.com/Green-Software-Foundation/sci-guide/tree/dev/use-case-submissions)。

GSF 鼓励应用 SCI 规范的团队[**提交使用案例**](https://github.com/Green-Software-Foundation/sci-guide/issues/new?assignees=atg-abhishek%2C+Henry-WattTime%2C+navveenb%2C+srini1978&labels=Case-study+submissions&projects=&template=case-study-template.md&title=Case-study+submissions)以供审阅。
