# OneText创始UX工程师

[](#onetext-founding-ux-engineer)

我们是[OneText](https://onetext.com)，一家由YC支持（2023年冬季）的湾区初创公司，我们正在寻找一位创始**UX工程师**来完全拥有并重新定义我们的仪表板和网页结账体验。

使用[此表格进行申请](https://forms.gle/SgXSewBMS1q7hkq48)

有任何问题请发送邮件至：[engineering@onetext.com](mailto:engineering@onetext.com)

这是[所有工程师在OneText的通用工作描述](https://gist.github.com/bluepnume/a6a7c793527067b770e49d2fa8cab3e2)

对于这个角色，我们希望找到一个可以充当*创始*设计和UX工程师的人。这意味着什么？

### 我们希望你能够拥有我们的UX

[](#we-want-you-to-take-ownership-of-our-ux)

整个团队对我们的平台应该如何发展有很多想法，这些是一个很好的起点，会帮助你在这个角色中快速起步。

但是：我们希望达到UX由*你的*愿景定义的地步，你愿意为此而战并完全拥有它。我们不喜欢由委员会设计的东西。

### 我们希望你重新定义我们最关键的体验：

[](#we-want-you-to-redefine-our-most-pivotal-experiences)

+   我们的商户仪表板，这是我们所有客户的入口，也是销售电话中展示的第一件事

+   我们的结账流程，允许每个消费者都通过OneText储存卡或PayPal账户并购买

+   我们的品牌消息体验（尤其是MMS和Whatsapp）

### 我们希望你能成为具有设计背景的UX工程师。

[](#we-want-you-to-be-a-ux-engineer-with-a-design-background)

我们知道我们在寻找一个独角兽。但我们也知道当设计团队交接给工程团队再交接给设计团队时，反馈循环有多么缓慢。

我们希望你能够提出一个出色的体验，并使之成为现实，无需中断你的流程或无休止的审查和批准。

你的工作流程无关紧要；你可能使用Figma或者直接在代码中设计。但我们需要你能够从零开始提出出色的用户体验，而不是按照预定义的设计或线框图进行工作。

### 我们希望你建立一些全新的东西

[](#we-want-to-you-build-something-new)

复制竞争对手现有平台是很诱人的。我们应该不断挑战这种诱惑，并建立真正独特且最优秀的客户体验，不受之前所做的限制。

这并不意味着我们不能从别人那里学习。我们可以并且应该学习。我们只是不应受它们的限制。

### 我们希望你成为一名出色的全方位工程师

[](#we-want-you-to-be-a-great-all-round-engineer)

显然，你的角色*主要*集中在设计和用户体验方面。所以基本要求是：

+   你在任何最适合你的工具中拥有坚实的设计背景。

+   你精通 React，并能编写出色的可重用组件（及其测试），能轻松组合成出色的体验。

+   你是 CSS（和/或 Tailwind）方面的专家，能够构建在任何设备上看起来和运行得很棒的设计和布局。

但是：我们只雇佣全栈工程师。这并不意味着你需要成为 Node 或数据库专家。但这确实意味着当有机会或需要时，你愿意去学习和查看 API，或者在构建完美用户体验时解除障碍。

### 我们希望你能带领整个工程团队迈向新的高度。

[](#we-want-you-to-level-up-the-entire-engineering-team)

任何涉及用户体验的 PR，我们希望你能以铁腕的方式参与其中 - 在发布之前进行审查，确保完美，并帮助我们提升水平。

我们希望你能把整个团队打造成卓越的 UX 工程师。

我们在面试时会问你的问题类型：

+   展示两到三个你为之自豪的经验和项目。

    +   如果你有另外6个月的时间来完善其中一个，你会如何利用这段时间？

+   你构建完美的 React 组件的规划和构建策略是什么？

    +   你如何判断大小是否合适？

    +   你如何处理状态管理？什么时候应该从原始 hooks 切换到像 Redux 这样的工具？

    +   你如何处理无障碍问题？

    +   你喜欢如何进行测试？

+   你如何看待 React？你喜欢什么，讨厌什么？

+   你如何看待 Tailwind？你喜欢什么，讨厌什么？

+   对于任何在线 Web 平台的仪表板，你最喜欢哪一个？可以选择多个。

    +   你对此有何看法？

    +   如果有机会，你会改变什么？

+   与我辩论。让我们选择一个我们意见不一的话题，进行深入讨论。
