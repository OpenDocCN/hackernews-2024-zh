# 白空间如何摧毁了一款企业应用（以及为什么数据密度很重要）

*宽敞。简约。清洁。* 丰富的白空间已经成为消费者应用程序中的事实设计美学。

我并不是来抨击这一趋势的。有效利用，白空间具有吸引力，并且可以极大地提升简单界面的可用性。白空间的丰富永远存在！

**那复杂的界面呢？** 企业软件设计师们知道我说的是哪些：技术支持控制面板、数据密集的物流系统和数字密集的会计系统。这些工具是我们的业务用户每天依赖的工具，用来完成他们的工作。

# 让我给你讲一个悲伤的故事。

主角是一位心怀善意的大型高科技公司的UX设计师，被分配了一个新项目：重新设计一个内部控制面板，它又丑又难学，每个屏幕上都塞满了内容（数据太多）。大家一致认为它需要现代化——毕竟看起来像是从2000年代初开始的！

因此，这位设计师着手解决问题，从现代消费者应用中汲取灵感。

新设计简化了每个屏幕。它将庞大的页面分解成更小、更专注的页面。它使用逐步显露来隐藏可能不重要的信息。而如今的用户并不介意滚动（咳咳），设计在所有常见的位置——标题周围、内容块中和表格行中——都融入了白空间。这些休息空间真是令人愉悦。

它在公司被迫退役前持续了一个月。

用户们非常讨厌这个新系统。当然，旧系统很丑陋，但他们需要的一切都近在咫尺！他们的工作非常快节奏——他们在一个技术支持呼叫中心工作，并根据生产力指标评分。他们没时间在时钟*不停走动*时点击或滚动查找信息。

在他们看来，这个新系统不是一次升级，而是一次失败。它不仅令人略感沮丧——*它让他们生气*。

# 这里的教训是什么？

一个大型企业自然拥有大规模的数据和通常与之直接交互的成千上万的用户——搜索、操作、报告等等。他们需要快速浏览这些数据，而不需要在界面中大费周章。

想想这样：就像你不会喜欢一页只有10个单词的词典（要翻好多页一样！），企业用户也不想要让他们为了找到他们要找的东西而费劲的系统。

但这并不意味着你的界面必须难看。

精心设计的数据密度可以在每个屏幕上呈现大量信息，同时保持清晰和易于扫描的内容层次结构。如果做得好，用户可以快速准确地访问他们需要的数据，而不会影响他们阅读文本或整体与应用程序交互的能力。

以下是如何增加数据密度而不降低应用程序现代美学的一些建议。

# 首先，尝试减少数据量。

在开始缩小字体大小以在每个屏幕上塞入更多数据之前，先看看所有这些数据是否真的有必要。问问自己：

+   **我能消除冗余数据吗？** 许多系统意外地以微小变化多次显示相同数据。（庞大的表格是查找重复信息的首要地点。）

+   **用户现在真的不需要看到的东西吗？** 如果你不知道，问一下！他们真的在使用他们现有的所有可用资源吗？

+   **我能更好地组织相关信息吗？** 并非每个离散的数据元素总是需要自己的表格单元格，即使在后端系统中是这样存储的。看看哪里可以通过合并元素来减少视觉复杂性 —— 比如将客户的名字和姓氏放在同一个字段中。

# 使用良好的排版标准。

> *“间距对于快速阅读长篇基本无意义的字符串（如序列号）至关重要，甚至对于较短的字符串（如电话号码和日期）也有帮助。”*
> 
> *– 罗伯特·布林赫斯特，《排版设计的要素》*

在企业应用程序中，适当的排版至关重要。字重、字间距和字体间距都会影响用户快速浏览信息的能力和速度。

+   **考虑在比较行之间的数字时使用等宽数字**。

+   **考虑货币的小数对齐**，当美元和美分很重要时。

+   **保持行高窄**，但要小心使用足够的空白来清晰分隔对象。

# 谨慎使用颜色。

尽可能减少次要内容的颜色。在设计中使用保守的调色板使您所使用的颜色更具冲击力，并能开放交互或视觉意义，例如在错误消息（通常为红色）或链接文本中。

在上述示例中，请注意灰色主题表格中单个红色突出显示的显著性。每行在水平扫描性方面稍有损失，但因增加了整体表格的扫描性而受益。考虑对您的应用程序更为重要的因素。

# 减少“装饰物”。

在讨论如何制作更易读表格时，罗伯特·布林赫斯特指出，

> *“在排版空间中，应该有最少量的装饰物（规则、框、点和其他用于穿越排版空间的导引物），有最多量的信息。”*
> 
> *– 罗伯特·布林赫斯特，《排版设计的要素》*

在增加应用程序密度时，请考虑页面上哪些元素最终是美观的，而不是有价值的内部逻辑和视觉结构。

# 让用户导出数据，而不是

有时你永远无法足够密集。当处理大数据集、长列表、复杂表格或一页又一页的结果时，有时最好的解决方案是让用户将数据带到另一个工具中，以便他们可以以不同的方式与之交互。

考虑为用户增加导出他们正在查看的数据（或其超集！如果你的导出可以包含比屏幕上能显示的*更多*数据？）的能力，可以通过 XML、XLS、JSON 或 CSV。

# 不要忘记触摸

除非你能确定可以安全地排除基于触摸的交互，不要忘记触摸目标的最小尺寸。例如，Material 在其无障碍页面上有关于触摸目标布局的良好[指南](https://material.io/design/usability/accessibility.html#layout-typography)，你一定要去查看。

# 关注 Material 的更新

在最近的 Material 更新中，Google 包含了[增加数据密度的新标准](https://www.material.io/design/layout/applying-density.html#usage)，这使得它对企业应用程序更加可用。他们提出了深思熟虑的具体建议，并附带了大量精彩的示例，展示密度如何影响你的布局。

# 总是功能优先于形式

企业设计师可以从消费者设计中获得很多好的启示，但功能性必须始终是我们的主要关注点。这是因为我们的用户没有选择，他们*必须*使用我们设计的工具——如果他们认为一个系统不可用，他们无法决定只是去下载另一个系统。因此，我们有责任将他们的日常生产力放在第一位——始终如此。
