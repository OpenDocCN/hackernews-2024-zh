# 欧盟用户如果在国外时间过长无法更新第三方iOS应用程序 • The Register

> 来源：[https://www.theregister.com/2024/03/06/apple_dma_geolocation/](https://www.theregister.com/2024/03/06/apple_dma_geolocation/)

看起来苹果确实在尽其所能让第三方iOS应用商店在欧盟新的合规规则下成为对任何希望利用它们的用户或开发者的痛处。

苹果发布了符合欧盟数字市场法规的iOS 17.4版本后不久（该法规要求苹果为用户提供更大的设备控制权），苹果还发布了一篇帮助页面，描述了它打算如何跟踪符合新权限的欧盟用户，以及如何关闭对在欧盟以外过多逗留的用户的访问权限。

除了明显需要在符合条件的欧盟国家注册的Apple ID（列在帮助页面上），苹果还将使用设备上的处理来收集“符合条件的指标”，该指标将被传输到苹果。该指标需要包括确定设备位置，但苹果表示实际上不会收集设备位置。

离开欧洲短期旅行的欧盟居民不会注意到任何影响，并将继续访问苹果称为“替代应用市场”和从这些市场安装的应用。但是，苹果表示，“如果你离开时间太长，将失去一些功能，包括安装新的替代应用市场。”

尚不清楚宽限期有多长，或者苹果何时会禁用欧盟居民在国外旅行期间的替代应用市场。苹果收集符合条件的指标的频率以及可能禁用的其他功能（例如，[非WebKit浏览器引擎](https://www.theregister.com/2024/01/25/apple_eu_dma/)）也没有明确说明。我们已经向苹果发出了一份声明请求，但尚未收到回复。

对于在国外逗留时间过长的替代应用市场用户可能会失去的东西，苹果并未提及删除应用市场或其应用程序，但确实表示“它们无法通过下载它们的市场更新”。

“如果管理不当，替代应用市场将给用户和开发者带来增加的隐私、安全和安全风险，”苹果指出，这与其宣布将遵守DMA的举措一样。具有讽刺意味的是，苹果在它的帮助页面上发表了这一声明，就在它说不允许欧盟居民从替代市场更新应用之后，这使用户容易受到它所说的符合DMA会引起的问题的影响。

Cupertino遵守DMA的字面意义，但并不符合精神，这一点无需进一步证明。®
