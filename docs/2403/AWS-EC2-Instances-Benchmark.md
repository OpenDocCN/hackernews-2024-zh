# AWS EC2 实例基准

> 来源：[https://runs-on.com/reference/benchmarks-ec2-instances/](https://runs-on.com/reference/benchmarks-ec2-instances/)

想知道哪种 EC2 实例类型具有最快的单线程 CPU 性能？您来对地方了。

AWS 有时会使用定制的 CPU 类型，您在像 Passmark 等网站上找不到它们。因此，如果您正在寻找最快的 CPU，选择哪一个可能有些困难。

在这个页面上，我们提供了大多数 EC2 实例类型的最新基准测试数据，跨以下区域：

+   北弗吉尼亚（美国东部-1）

+   俄勒冈（美国西部-2）

+   爱尔兰（欧洲西部-1）

+   法兰克福（欧洲中部-1）

这可以帮助您根据处理器型号、架构和单线程 CPU 速度做出选择最佳工作流的明智决策。

由于速度并非一切，基准还列出了每种实例类型的按需和抢占定价，以及随时间变化的趋势。

最后，当您运行工作流时，可靠性至关重要，因此我们为每种实例类型提供中断百分比的数据，让您清楚地了解稳定性预期。

像往常一样，最好自己进行基准测试，但这至少可以为您指出正确方向的一些线索。

注：所有实例类型均使用 2 vCPU 变体启动。

## 分析

### 最快的 AWS EC2 x86_64 实例是什么？

对于x86_64架构，最快的EC2实例因具体工作负载和AWS区域而异。然而，来自计算优化（C7a, C7i, m7a, m7i, r7a, r7iz）系列的实例配备了最快的CPU。

### 最快的AWS EC2 ARM64实例是什么？

对于ARM64架构，AWS Graviton系列的实例（如C7g, M7g）旨在提供最佳性能。这些实例针对多种工作负载进行了优化，并为针对ARM64构建的应用程序提供了显著的价格性能优势。
